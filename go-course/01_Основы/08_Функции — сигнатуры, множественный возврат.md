## 📌 Теория

### 8.1 Базовый синтаксис функции

```go
func add(a int, b int) int {
    return a + b
}
```

Структура:

- `func` — ключевое слово
- `add` — имя
- `(a int, b int)` — параметры с типами
- `int` — тип возвращаемого значения
- тело в `{}`

> ⚠️ В Go тип пишется **после** имени — `a int`, а не `int a` как в C/Java. Это касается и переменных, и параметров, и возвратов.

---

### 8.2 Сокращение типов параметров

Если несколько параметров подряд имеют один тип — можно указать тип один раз:

```go
// Полная форма
func add(a int, b int) int { return a + b }

// Сокращённая — тип указан раз для группы
func add(a, b int) int { return a + b }

// Смешанные типы
func mix(a, b int, name string, c, d float64) {}
```

### 8.3 Множественный возврат — фишка Go

Функция может возвращать **несколько значений**. Это одна из самых характерных черт Go:

```go
func divmod(a, b int) (int, int) {
    return a / b, a % b
}

func main() {
    q, r := divmod(17, 5)
    fmt.Println(q, r)  // 3 2
}
```

Самое частое применение — возврат **результата и ошибки**:

```go
func parseAge(s string) (int, error) {
    age, err := strconv.Atoi(s)
    if err != nil {
        return 0, err
    }
    return age, nil
}
```

> 💡 Это центральный паттерн Go: вместо исключений функция возвращает `(результат, error)`. Ты уже встречал его с `strconv.Atoi`. Подробно про ошибки — в блоке ООП (уроки 22-24).

### 8.4 Именованные возвращаемые значения

Можно дать возвращаемым значениям имена. Тогда они работают как заранее объявленные переменные:

```go
func divmod(a, b int) (q int, r int) {
    q = a / b   // присваиваем именованным переменным
    r = a % b
    return      // "голый" return — вернёт q и r автоматически
}
```

`return` без аргументов (его называют **naked return**) возвращает текущие значения именованных переменных.

> ⚠️ **Осторожно с naked return:** в коротких функциях это удобно и читаемо. В длинных — снижает читаемость (непонятно что возвращается). Идиома: используй именованные возвраты для документирования, но `return q, r` явно в длинных функциях.

Полезное применение — именованные возвраты + defer (см. урок 9) для обработки ошибок:

```go
func process() (result int, err error) {
    defer func() {
        if r := recover(); r != nil {
            err = fmt.Errorf("паника: %v", r)
        }
    }()
    // ...
    return 42, nil
}
```

### 8.5 Variadic функции (переменное число аргументов)

`...T` означает "ноль или больше аргументов типа T". Внутри функции это срез `[]T`:

```go
func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

func main() {
    fmt.Println(sum(1, 2, 3))        // 6
    fmt.Println(sum(1, 2, 3, 4, 5))  // 15
    fmt.Println(sum())               // 0
}
```

> 💡 Ты постоянно используешь variadic функцию — это `fmt.Println(a, b, c, ...)`. Она принимает любое число аргументов.

#### Распаковка среза в variadic

Если у тебя уже есть срез, его можно "распаковать" через `...`:

```go
nums := []int{1, 2, 3, 4}
fmt.Println(sum(nums...))  // распаковка среза в аргументы → 10
```

> ⚠️ Variadic параметр должен быть **последним** в списке параметров:

```go
func f(prefix string, nums ...int) {}  // ✅
func f(nums ...int, suffix string) {}   // ❌ ошибка
```


### 8.7 Передача аргументов — всегда по значению

В Go **все аргументы передаются по значению** (копируются). Это важно понять:

```go
func increment(x int) {
    x++  // меняется КОПИЯ, оригинал не тронут
}

func main() {
    n := 5
    increment(n)
    fmt.Println(n)  // 5, не 6!
}
```

Чтобы изменить оригинал — передавай **указатель** (подробно в уроке 11):

```go
func increment(x *int) {
    *x++  // меняем значение по адресу
}

func main() {
    n := 5
    increment(&n)
    fmt.Println(n)  // 6
}
```

> ⚠️ **Важный нюанс:** срезы, карты и каналы — это ссылочные типы. Даже передаваясь "по значению", они содержат указатель на внутренние данные. Поэтому изменение элементов среза внутри функции **видно снаружи**. Это частый источник путаницы — разберём детально в уроках 11, 13, 14.

```go
func modify(s []int) {
    s[0] = 999  // это изменение ВИДНО снаружи!
}
```

### 8.8 Функция main и init

```go
package main

// init вызывается автоматически ДО main, при инициализации пакета
func init() {
    fmt.Println("инициализация")
}

func main() {
    fmt.Println("основная программа")
}
```

> 💡 `init()` — особая функция, вызывается автоматически при загрузке пакета, до `main`. Используется для настройки. Их может быть несколько (даже в одном файле). Не вызывается вручную.

## 💡 Примеры

### Пример 1 — Множественный возврат с ошибкой

```go
package main

import (
    "fmt"
    "strconv"
)

func safeDivide(a, b int) (int, error) {
    if b == 0 {
        return 0, fmt.Errorf("деление на ноль")
    }
    return a / b, nil
}

func main() {
    if result, err := safeDivide(10, 2); err != nil {
        fmt.Println("ошибка:", err)
    } else {
        fmt.Println("результат:", result)
    }

    if _, err := safeDivide(10, 0); err != nil {
        fmt.Println("ошибка:", err)
    }

    _ = strconv.Itoa(5)  // просто чтобы показать импорт
}
```

### Пример 2 — Variadic + распаковка

```go
package main

import "fmt"

func max(first int, rest ...int) int {
    m := first
    for _, n := range rest {
        if n > m {
            m = n
        }
    }
    return m
}

func main() {
    fmt.Println(max(3, 7, 2, 9, 4))  // 9

    nums := []int{5, 1, 8, 3}
    fmt.Println(max(nums[0], nums[1:]...))  // распаковка хвоста среза
}
```

### Пример 3 — Именованные возвраты

```go
package main

import "fmt"

func minMax(a, b, c int) (min, max int) {
    min, max = a, a
    if b < min { min = b }
    if c < min { min = c }
    if b > max { max = b }
    if c > max { max = c }
    return  // naked return — вернёт min, max
}

func main() {
    lo, hi := minMax(5, 2, 8)
    fmt.Printf("min=%d max=%d\n", lo, hi)  // min=2 max=8
}
```

### Пример 4 — Передача по значению vs указатель

```go
package main

import "fmt"

func tryChangeValue(x int)  { x = 100 }
func changeViaPointer(x *int) { *x = 100 }

func main() {
    n := 5

    tryChangeValue(n)
    fmt.Println(n)  // 5 — копия изменилась, оригинал нет

    changeViaPointer(&n)
    fmt.Println(n)  // 100 — изменили через указатель
}
```

## ⚠️ Подводные камни

### 1. Тип после имени

```go
func f(int x) {}   // ❌
func f(x int) {}   // ✅
```

### 2. Аргументы копируются (для базовых типов)

```go
func f(x int) { x = 10 }
n := 5
f(n)  // n всё ещё 5
```

### 3. Variadic только последний параметр

```go
func f(nums ...int, s string) {}  // ❌
func f(s string, nums ...int) {}  // ✅
```

### 4. Naked return в длинных функциях — плохо

```go
// В длинной функции непонятно что вернётся
func longFunc() (result int, err error) {
    // ...50 строк...
    return  // что именно? приходится искать
}
```

### 5. Нельзя игнорировать возвращённую ошибку молча

```go
result, _ := mayFail()  // технически можно, но это запах кода
// линтеры (errcheck) ругаются на игнор ошибок
```

## 📋 Краткое резюме

- Тип пишется после имени: `a int`; для группы одного типа — `a, b int`.
- Множественный возврат — фишка Go; центральный паттерн `(результат, error)` вместо исключений.
- Именованные возвраты + naked `return`: удобно в коротких функциях, вредно в длинных.
- Variadic `...T` = срез внутри; распаковка среза через `slice...`; параметр всегда последний.
- Аргументы передаются по значению (копируются); чтобы менять оригинал — указатель.
- Ссылочные типы (срез, карта, канал) содержат указатель — изменение элементов видно снаружи.
- `init()` вызывается автоматически до `main` при инициализации пакета.
