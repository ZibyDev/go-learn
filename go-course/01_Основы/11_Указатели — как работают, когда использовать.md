# Урок 11 — Указатели: как работают, когда использовать

## 📌 Теория

### 11.1 Что такое указатель

Указатель — это переменная, которая хранит **адрес** другой переменной в памяти, а не само значение. Если переменная — это "коробка со значением", то указатель — это "записка с адресом коробки".

```go
x := 42
p := &x        // p хранит адрес x
fmt.Println(x)  // 42      — значение
fmt.Println(p)  // 0xc0000... — адрес (шестнадцатеричный)
fmt.Println(*p) // 42      — значение по адресу (разыменование)
```

> 💡 Если ты работал с C/C++ — концепция знакома, но в Go указатели **безопаснее**: нет арифметики указателей (нельзя `p++`), есть сборщик мусора, нет висячих указателей.

---

### 11.2 Два оператора: & и *

|Оператор|Название|Что делает|
|---|---|---|
|`&x`|взятие адреса|возвращает указатель на `x`|
|`*p`|разыменование|возвращает значение по адресу `p`|

```go
x := 10

p := &x         // & — берём адрес x, p имеет тип *int
fmt.Println(*p) // * — читаем значение по адресу → 10

*p = 20         // * слева — записываем значение по адресу
fmt.Println(x)  // 20 — изменили x через указатель!
```

> 💡 Запомни направление: `&` превращает значение в адрес (value → pointer), `*` превращает адрес в значение (pointer → value). Они противоположны.

---

### 11.3 Тип указателя

Тип указателя записывается как `*T` — "указатель на T":

```go
var p *int       // указатель на int
var s *string    // указатель на string
var pp **int     // указатель на указатель на int (редко)
```

> ⚠️ Zero value указателя — `nil`. Разыменование nil-указателя вызывает панику:

```go
var p *int
// fmt.Println(*p)  // panic: nil pointer dereference
if p != nil {
    fmt.Println(*p)
}
```

---

### 11.4 Зачем нужны указатели — причина 1: изменение через функции

Из урока 8 ты помнишь: аргументы передаются **по значению** (копируются). Указатель позволяет функции изменить оригинал:

```go
func incrementValue(x int) {
    x++  // меняется копия
}

func incrementPointer(p *int) {
    *p++  // меняется оригинал по адресу
}

func main() {
    n := 5

    incrementValue(n)
    fmt.Println(n)  // 5 — без изменений

    incrementPointer(&n)
    fmt.Println(n)  // 6 — изменили!
}
```

---

### 11.5 Зачем нужны указатели — причина 2: избежать копирования больших структур

Когда передаёшь большую структуру по значению, она целиком копируется. Указатель передаёт только адрес (8 байт на 64-битной системе):

```go
type BigStruct struct {
    data [1000]int  // массив на 1000 элементов
}

// Копирует все 1000 элементов при каждом вызове
func processValue(s BigStruct) {}

// Передаёт только адрес — дёшево
func processPointer(s *BigStruct) {}
```

> 💡 Это станет особенно важно в уроке 16 (методы): выбор receiver по значению или по указателю во многом определяется размером структуры и необходимостью её менять.

---

### 11.6 new — создание указателя

Функция `new(T)` выделяет память под значение типа `T`, инициализирует его zero value и возвращает указатель:

```go
p := new(int)    // p типа *int, *p == 0
fmt.Println(*p)  // 0

*p = 42
fmt.Println(*p)  // 42
```

`new(int)` эквивалентно:

```go
var x int
p := &x
```

> 💡 На практике `new` используется редко. Чаще указатель получают через `&` от уже существующей переменной или от литерала структуры (см. ниже). `new` чаще встречается в дженерик-коде.

---

### 11.7 Указатели на структуры (забегаем вперёд)

Указатели чаще всего используются со структурами. Go позволяет обращаться к полям через указатель **без явного разыменования**:

```go
type Point struct {
    X, Y int
}

func main() {
    p := &Point{X: 1, Y: 2}  // указатель на структуру

    // Go автоматически разыменовывает: (*p).X можно писать как p.X
    fmt.Println(p.X)  // 1 — не нужно (*p).X
    p.Y = 20          // тоже работает напрямую
    fmt.Println(p.Y)  // 20
}
```

> 💡 Это синтаксический сахар: `p.X` для указателя автоматически означает `(*p).X`. Структуры подробно в уроке 15, здесь просто показываю что указатели с ними дружат. В C для этого нужен оператор `->`, в Go — точка работает и для значения, и для указателя.

---

### 11.8 Возврат указателя на локальную переменную — это безопасно!

В C/C++ возврат адреса локальной переменной — классическая ошибка (висячий указатель). В Go это **безопасно** благодаря escape analysis: компилятор сам решает разместить переменную в куче, если на неё возвращается указатель.

```go
func newCounter() *int {
    count := 0     // локальная переменная
    return &count  // возвращаем её адрес — в Go это ОК!
}

func main() {
    p := newCounter()
    *p = 100
    fmt.Println(*p)  // 100 — переменная жива
}
```

> 💡 Это работает потому, что Go-компилятор анализирует "убегает" ли переменная за пределы функции (escape analysis). Если да — размещает в куче, и сборщик мусора освободит её когда никто не будет ссылаться. Подробно в уроке 54.

---

### 11.9 Ссылочные типы уже содержат указатели

Важный нюанс из урока 8: срезы, карты и каналы — это **ссылочные типы**. Внутри они уже содержат указатель на данные. Поэтому их не нужно передавать как `*[]int` — изменение элементов и так видно:

```go
func modifySlice(s []int) {
    s[0] = 999  // видно снаружи, т.к. срез ссылается на те же данные
}

func main() {
    nums := []int{1, 2, 3}
    modifySlice(nums)
    fmt.Println(nums)  // [999 2 3]
}
```

> ⚠️ Но это касается только **изменения элементов**. Если внутри функции сделать `s = append(s, ...)` или `s = nil` — оригинал не изменится, потому что меняется локальная копия "заголовка" среза. Это тонкость, которую разберём детально в уроке 13.

---

### 11.10 Когда использовать указатели

|Используй указатель|Используй значение|
|---|---|
|Нужно изменить аргумент внутри функции|Аргумент только читается|
|Большая структура (дорого копировать)|Маленькая структура (int, небольшой struct)|
|Нужно отличать "нет значения" (nil) от zero value|Значение всегда есть|
|Методы, меняющие получатель (урок 16)|Неизменяемые операции|

> 💡 Правило для начала: если сомневаешься и структура небольшая — передавай по значению (это безопаснее, нет nil-паник). Указатели — когда есть конкретная причина: изменение или размер.

## 💡 Примеры

### Пример 1 — Базовая работа с указателем

```go
package main

import "fmt"

func main() {
    x := 42
    p := &x

    fmt.Println("значение x:", x)    // 42
    fmt.Println("адрес x:", p)       // 0x...
    fmt.Println("значение по p:", *p) // 42

    *p = 100
    fmt.Println("x после *p=100:", x) // 100
}
```

### Пример 2 — Swap через указатели

```go
package main

import "fmt"

func swap(a, b *int) {
    *a, *b = *b, *a
}

func main() {
    x, y := 1, 2
    swap(&x, &y)
    fmt.Println(x, y)  // 2 1
}
```

### Пример 3 — Указатель отличает nil от нуля

```go
package main

import "fmt"

// Возвращаем *int: nil означает "не найдено", иначе указатель на значение
func findValue(found bool) *int {
    if !found {
        return nil
    }
    v := 42
    return &v
}

func main() {
    if p := findValue(true); p != nil {
        fmt.Println("найдено:", *p)
    }

    if p := findValue(false); p == nil {
        fmt.Println("не найдено")
    }
}
```

### Пример 4 — Изменение через указатель в функции

```go
package main

import "fmt"

func reset(p *int) {
    *p = 0
}

func addBonus(score *int, bonus int) {
    *score += bonus
}

func main() {
    health := 100
    addBonus(&health, 50)
    fmt.Println(health)  // 150

    reset(&health)
    fmt.Println(health)  // 0
}
```

## ⚠️ Подводные камни

### 1. Разыменование nil-указателя — паника

```go
var p *int
*p = 10  // panic: nil pointer dereference
```

### 2. Нет арифметики указателей

```go
p := &x
// p++       // ❌ в Go нельзя двигать указатель (в отличие от C)
// p + 1     // ❌
```

### 3. & нельзя брать от литерала базового типа

```go
p := &42      // ❌ нельзя взять адрес константы/литерала
x := 42
p := &x       // ✅
// Но от литерала СТРУКТУРЫ — можно: &Point{1,2}
```

### 4. Путаница * в объявлении и в выражении

```go
var p *int   // здесь * — часть ТИПА (указатель на int)
v := *p      // здесь * — операция разыменования
```

### 5. Изменение копии указателя не меняет оригинал

```go
func reassign(p *int) {
    x := 99
    p = &x  // меняем КУДА смотрит локальная копия p, оригинал не тронут
}
// чтобы изменить значение — *p = 99, а не p = &x
```

## 📋 Краткое резюме

- Указатель хранит адрес переменной; `&` берёт адрес (value→pointer), `*` разыменовывает (pointer→value).
- Тип `*T`, zero value = `nil`; разыменование nil — паника.
- Две причины использовать: изменить аргумент в функции и избежать копирования больших структур.
- `new(T)` выделяет память и возвращает указатель (на практике редко).
- У указателя на структуру поля доступны напрямую: `p.X` вместо `(*p).X`.
- Возврат адреса локальной переменной в Go **безопасен** (escape analysis), в отличие от C.
- Нет арифметики указателей (`p++` нельзя).
- Ссылочные типы (срез, карта) уже содержат указатель внутри.
