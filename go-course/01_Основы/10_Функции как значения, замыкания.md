## 📌 Теория

### 10.1 Функции — это значения первого класса

В Go функции — полноценные значения. Их можно присваивать переменным, передавать в другие функции и возвращать из функций. Если ты работал с JS или Kotlin — это знакомо.

```go
func add(a, b int) int { return a + b }

func main() {
    f := add          // присваиваем функцию переменной
    result := f(3, 4) // вызываем через переменную
    fmt.Println(result) // 7
}
```

Тип переменной `f` здесь — `func(int, int) int`. Сигнатура функции и есть её тип.

---

### 10.2 Тип функции

Тип функции описывается её сигнатурой: параметры и возвраты.

```go
var operation func(int, int) int  // переменная типа "функция (int,int) -> int"

operation = func(a, b int) int { return a * b }
fmt.Println(operation(3, 4))  // 12

operation = func(a, b int) int { return a - b }
fmt.Println(operation(10, 3)) // 7
```

> 💡 Zero value функционального типа — `nil`. Вызов nil-функции вызовет панику, так что проверяй перед вызовом если функция могла остаться неинициализированной.

```go
var f func()
// f()  // panic: nil pointer
if f != nil {
    f()
}
```

---

### 10.3 Анонимные функции

Функцию можно объявить без имени — прямо на месте:

```go
// Присвоить переменной
square := func(x int) int {
    return x * x
}
fmt.Println(square(5))  // 25

// Вызвать сразу (IIFE — immediately invoked)
result := func(x int) int {
    return x * x
}(5)
fmt.Println(result)  // 25
```

Ты уже использовал анонимные функции в `defer func(){...}()` из урока 9.

---

### 10.4 Функции как аргументы (callbacks)

Функцию можно передать в другую функцию. Это основа для гибких, переиспользуемых API:

```go
func applyTo(nums []int, op func(int) int) []int {
    result := make([]int, len(nums))  // срез — подробно в уроке 13
    for i, n := range nums {
        result[i] = op(n)
    }
    return result
}

func main() {
    nums := []int{1, 2, 3, 4}

    doubled := applyTo(nums, func(x int) int { return x * 2 })
    fmt.Println(doubled)  // [2 4 6 8]

    squared := applyTo(nums, func(x int) int { return x * x })
    fmt.Println(squared)  // [1 4 9 16]
}
```

> 💡 Это паттерн "стратегия": поведение передаётся как параметр. В стандартной библиотеке так работает, например, `sort.Slice` — ты передаёшь функцию сравнения.

### 10.5 Функции как возвращаемые значения

Функция может вернуть другую функцию:

```go
func multiplier(factor int) func(int) int {
    return func(x int) int {
        return x * factor
    }
}

func main() {
    double := multiplier(2)
    triple := multiplier(3)

    fmt.Println(double(5))  // 10
    fmt.Println(triple(5))  // 15
}
```

Обрати внимание: возвращённая функция "помнит" значение `factor`. Это и есть замыкание.

### 10.6 Замыкания (closures) — главная тема урока

**Замыкание** — это функция, которая "захватывает" переменные из окружающей области видимости. Она продолжает иметь к ним доступ даже после того, как внешняя функция завершилась.

```go
func counter() func() int {
    count := 0          // эта переменная "захвачена" замыканием
    return func() int {
        count++         // замыкание изменяет захваченную переменную
        return count
    }
}

func main() {
    next := counter()
    fmt.Println(next())  // 1
    fmt.Println(next())  // 2
    fmt.Println(next())  // 3

    // Новый счётчик — своя независимая count
    other := counter()
    fmt.Println(other()) // 1 — независим от next
}
```

> 💡 Ключевой момент: `count` не уничтожается после выхода из `counter()`, потому что возвращённое замыкание всё ещё на неё ссылается. Каждый вызов `counter()` создаёт **новую** независимую `count`.

---

### 10.7 Захват по ссылке, не по значению

Замыкание захватывает **саму переменную**, а не её копию. Это значит, что изменения видны:

```go
func main() {
    x := 10

    show := func() {
        fmt.Println("x =", x)
    }

    show()       // x = 10
    x = 20
    show()       // x = 20 — замыкание видит новое значение!
}
```

> ⚠️ Это и есть причина классической "ловушки цикла" из урока 7. До Go 1.22 все замыкания в цикле захватывали **одну и ту же** переменную цикла:

```go
// До Go 1.22 — все три функции печатали 3
funcs := make([]func(), 3)
for i := 0; i < 3; i++ {
    funcs[i] = func() { fmt.Println(i) }  // все захватили одну i
}
// На Go 1.22+ — каждая итерация своя i, выведет 0,1,2
```

Старый способ обойти (до 1.22) — создать локальную копию:

```go
for i := 0; i < 3; i++ {
    i := i  // новая переменная на каждой итерации
    funcs[i] = func() { fmt.Println(i) }
}
```

### 10.8 Практические применения замыканий

#### Конфигурируемые функции

```go
func makeValidator(min, max int) func(int) bool {
    return func(value int) bool {
        return value >= min && value <= max
    }
}

inRange := makeValidator(1, 100)
fmt.Println(inRange(50))   // true
fmt.Println(inRange(150))  // false
```

#### Накопление состояния

```go
func accumulator() func(int) int {
    sum := 0
    return func(x int) int {
        sum += x
        return sum
    }
}

acc := accumulator()
fmt.Println(acc(10))  // 10
fmt.Println(acc(20))  // 30
fmt.Println(acc(5))   // 35
```

### 10.9 Рекурсивные анонимные функции

Чтобы анонимная функция вызывала сама себя, её нужно сначала объявить как переменную:

```go
var fib func(int) int
fib = func(n int) int {
    if n < 2 {
        return n
    }
    return fib(n-1) + fib(n-2)
}
fmt.Println(fib(10))  // 55
```

> 💡 Нужны две строки: `var fib func(int) int` объявляет имя, затем присваивание. Если написать `fib := func...` в одну строку — внутри тела имя `fib` ещё не будет видно.

## 💡 Примеры

### Пример 1 — Функция в переменной и как аргумент

```go
package main

import "fmt"

func operate(a, b int, op func(int, int) int) int {
    return op(a, b)
}

func main() {
    add := func(x, y int) int { return x + y }
    mul := func(x, y int) int { return x * y }

    fmt.Println(operate(3, 4, add))  // 7
    fmt.Println(operate(3, 4, mul))  // 12

    // Анонимная прямо в вызове
    fmt.Println(operate(10, 3, func(x, y int) int { return x - y }))  // 7
}
```

### Пример 2 — Замыкание-счётчик

```go
package main

import "fmt"

func makeCounter() func() int {
    count := 0
    return func() int {
        count++
        return count
    }
}

func main() {
    c1 := makeCounter()
    c2 := makeCounter()

    fmt.Println(c1())  // 1
    fmt.Println(c1())  // 2
    fmt.Println(c2())  // 1 — независимый
    fmt.Println(c1())  // 3
}
```

### Пример 3 — Фабрика функций

```go
package main

import "fmt"

func powerOf(exponent int) func(int) int {
    return func(base int) int {
        result := 1
        for i := 0; i < exponent; i++ {
            result *= base
        }
        return result
    }
}

func main() {
    square := powerOf(2)
    cube := powerOf(3)

    fmt.Println(square(5))  // 25
    fmt.Println(cube(2))    // 8
}
```

### Пример 4 — Захват переменной по ссылке

```go
package main

import "fmt"

func main() {
    message := "привет"

    printer := func() {
        fmt.Println(message)
    }

    printer()              // привет
    message = "пока"
    printer()              // пока — видит изменение
}
```

## ⚠️ Подводные камни

### 1. nil-функция паникует при вызове

```go
var f func()
f()  // panic: nil pointer dereference
```

### 2. Замыкание захватывает переменную, а не значение

```go
x := 1
f := func() { fmt.Println(x) }
x = 2
f()  // 2, не 1
```

### 3. Ловушка переменной цикла (до Go 1.22)

```go
// На старых версиях все замыкания делят одну переменную цикла
for i := 0; i < 3; i++ {
    defer func() { fmt.Println(i) }()  // до 1.22: 3,3,3; на 1.22+: 2,1,0
}
```

### 4. Рекурсивная анонимка требует предварительного объявления

```go
f := func(n int) int { return f(n-1) }  // ❌ f не видна внутри себя
// Нужно:
var f func(int) int
f = func(n int) int { return f(n-1) }   // ✅
```

### 5. Захват в горутинах (забегаем вперёд)

```go
// Передавай переменную как аргумент, а не захватывай — детали в блоке concurrency
go func(val int) { fmt.Println(val) }(i)
```

## 📋 Краткое резюме

- Функции — значения первого класса: присваиваются, передаются, возвращаются.
- Тип функции = её сигнатура (`func(int, int) int`); zero value = `nil` (вызов nil паникует).
- Анонимные функции и IIFE (`func(){...}()`); callbacks как параметры — паттерн «стратегия».
- Замыкание захватывает переменную из окружения **по ссылке**, а не копию — изменения видны.
- Каждый вызов функции-фабрики создаёт независимое замыкание со своим состоянием.
- Несколько функций могут разделять одно общее захваченное состояние.
- Рекурсивная анонимка требует предварительного `var f func(...)` перед присваиванием.
