## 📌 Теория

### 12.1 Что такое массив

Массив в Go — это **фиксированная по длине** последовательность элементов одного типа. Длина — часть типа массива и задаётся при объявлении.

```go
var arr [5]int           // массив из 5 int, все = 0 (zero value)
fmt.Println(arr)          // [0 0 0 0 0]

primes := [5]int{2, 3, 5, 7, 11}  // с инициализацией
fmt.Println(primes)        // [2 3 5 7 11]
```

> ⚠️ **Главное:** длина массива неизменна. `[5]int` и `[10]int` — это **разные типы**. Нельзя присвоить один другому, нельзя «дорастить» массив.

---

### 12.2 Способы объявления

```go
// Полное с указанием длины
var a [3]int = [3]int{1, 2, 3}

// С выводом типа
b := [3]int{1, 2, 3}

// Длина выводится из количества элементов через ...
c := [...]int{1, 2, 3, 4}  // компилятор сам посчитает: [4]int
fmt.Println(len(c))         // 4

// Инициализация конкретных индексов
d := [5]int{0: 10, 4: 50}   // [10 0 0 0 50]

// Многомерный массив
var grid [3][3]int          // 3x3, все нули
grid[1][1] = 5
```

---

### 12.3 Длина — часть типа

Это фундаментальное отличие от большинства языков:

```go
var a [3]int
var b [4]int
// a = b  // ❌ ошибка: cannot use b (type [4]int) as type [3]int

func process(arr [3]int) {}  // принимает ТОЛЬКО [3]int
// process(b)  // ❌ [4]int не подойдёт
```

> 💡 Именно из-за этой жёсткости массивы напрямую используются редко. Чаще используют **срезы** (урок 13), которые гибче. Но понять массивы нужно — срезы построены поверх них.

---

### 12.4 Массивы копируются по значению — КЛЮЧЕВОЙ момент

В отличие от многих языков (где массив — ссылка), в Go массив — это **значение**. При присваивании или передаче в функцию он **копируется целиком**:

```go
a := [3]int{1, 2, 3}
b := a          // b — ПОЛНАЯ КОПИЯ a
b[0] = 999
fmt.Println(a)  // [1 2 3] — оригинал не изменился!
fmt.Println(b)  // [999 2 3]
```

То же с функциями:

```go
func modify(arr [3]int) {
    arr[0] = 999  // меняется КОПИЯ
}

func main() {
    a := [3]int{1, 2, 3}
    modify(a)
    fmt.Println(a)  // [1 2 3] — без изменений!
}
```

> ⚠️ Это противоположно поведению срезов, карт (ссылочные типы) и массивов в C/Python/JS. Если нужно изменить массив в функции — передавай указатель `*[3]int` или используй срез.

```go
func modifyPtr(arr *[3]int) {
    arr[0] = 999  // через указатель меняем оригинал
}
```

---

### 12.5 Сравнение массивов

Массивы можно сравнивать через `==`, если их элементы сравнимы. Сравниваются поэлементно:

```go
a := [3]int{1, 2, 3}
b := [3]int{1, 2, 3}
c := [3]int{1, 2, 4}

fmt.Println(a == b)  // true — все элементы равны
fmt.Println(a == c)  // false
```

> 💡 Это работает только для массивов **одного типа** (одинаковой длины). Срезы, кстати, через `==` сравнивать **нельзя** (только с nil) — ещё одно отличие.

---

### 12.6 Перебор массива

```go
arr := [4]int{10, 20, 30, 40}

// Через индекс
for i := 0; i < len(arr); i++ {
    fmt.Println(i, arr[i])
}

// Через range
for i, v := range arr {
    fmt.Println(i, v)
}
```

> 💡 `range` по массиву тоже копирует элементы в `v`. Для больших массивов или если нужно менять — обращайся по индексу `arr[i]`.

---

### 12.7 Массив vs срез — краткое сравнение

|Свойство|Массив `[N]T`|Срез `[]T`|
|---|---|---|
|Длина|Фиксированная, часть типа|Динамическая|
|Передача|Копируется по значению|Ссылается на данные|
|Сравнение `==`|Можно|Нельзя (только с nil)|
|Объявление|`[5]int`|`[]int`|
|Применение|Редко напрямую|Основная рабочая коллекция|
|Изменение в функции|Нужен указатель|Видно сразу|

> 💡 На практике 95% времени ты работаешь со **срезами**. Массивы нужны когда: размер строго фиксирован и известен (например, `[16]byte` для хэша), нужно значение-копия, или для производительности (массив на стеке без аллокаций). Подробно срезы — следующий урок.

### 12.8 Связь массива и среза (мостик к уроку 13)

Срез — это «окно» в массив. Из массива можно получить срез:

```go
arr := [5]int{1, 2, 3, 4, 5}
s := arr[1:4]   // срез, ссылающийся на элементы arr[1], arr[2], arr[3]
fmt.Println(s)  // [2 3 4]

s[0] = 999      // меняем через срез
fmt.Println(arr) // [1 999 3 4 5] — изменился МАССИВ!
```

> 💡 Это ключевая связь: срез не хранит данные сам, он ссылается на массив. Поэтому изменение через срез видно в массиве. Детально — в уроке 13.

## 💡 Примеры

### Пример 1 — Массив копируется

```go
package main

import "fmt"

func main() {
    original := [3]string{"a", "b", "c"}
    copy := original     // полная копия

    copy[0] = "X"

    fmt.Println("оригинал:", original)  // [a b c]
    fmt.Println("копия:", copy)         // [X b c]
}
```

### Пример 2 — Сумма элементов массива

```go
package main

import "fmt"

func sum(arr [5]int) int {
    total := 0
    for _, v := range arr {
        total += v
    }
    return total
}

func main() {
    nums := [5]int{10, 20, 30, 40, 50}
    fmt.Println(sum(nums))  // 150
}
```

### Пример 3 — Изменение через указатель на массив

```go
package main

import "fmt"

func fillWithIndex(arr *[5]int) {
    for i := range arr {
        arr[i] = i * 10
    }
}

func main() {
    var data [5]int
    fillWithIndex(&data)
    fmt.Println(data)  // [0 10 20 30 40]
}
```

### Пример 4 — Сравнение массивов

```go
package main

import "fmt"

func main() {
    a := [3]int{1, 2, 3}
    b := [3]int{1, 2, 3}
    c := [3]int{3, 2, 1}

    fmt.Println(a == b)  // true
    fmt.Println(a == c)  // false
}
```

## ⚠️ Подводные камни

### 1. Длина — часть типа

```go
var a [3]int
var b [5]int
// a = b  // ❌ разные типы
```

### 2. Массив копируется при передаче

```go
func f(arr [3]int) { arr[0] = 9 }  // меняет копию
a := [3]int{1, 2, 3}
f(a)  // a не изменится
```

### 3. range копирует элементы

```go
arr := [3]int{1, 2, 3}
for _, v := range arr {
    v = v * 2  // меняет копию v, массив не трогается
}
// arr всё ещё [1 2 3]
```

### 4. Срезы нельзя сравнить через ==

```go
s1 := []int{1, 2}
s2 := []int{1, 2}
// s1 == s2  // ❌ ошибка компиляции (можно только s1 == nil)
```

### 5. Большие массивы дороги при копировании

```go
func f(arr [1000000]int) {}  // копирует миллион элементов при каждом вызове!
// Используй срез или указатель
```

## 📋 Краткое резюме

- Массив — фиксированная длина, которая является **частью типа**: `[3]int` ≠ `[5]int`.
- Объявление: `[5]int{...}`, `[...]int{...}` (длина по элементам), по индексам `[5]int{0:10, 4:50}`.
- Массив **копируется по значению** при присваивании и передаче в функцию (в отличие от среза).
- Чтобы изменить массив в функции — передавай указатель `*[N]T`.
- Массивы сравнимы через `==` (поэлементно); срезы — нельзя (только с nil).
- `range` по массиву копирует элементы в переменную значения.
- Срез — «окно» в массив: изменение через срез видно в исходном массиве (мостик к уроку 13).
