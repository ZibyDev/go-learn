## 📌 Теория

### 14.1 Что такое карта

Карта (map) — это структура «ключ-значение» (хеш-таблица). Аналог `dict` в Python, `HashMap` в Java, объекта в JS. Хранит пары: по уникальному ключу — значение.

```go
ages := map[string]int{
    "Alice": 30,
    "Bob":   25,
}
fmt.Println(ages["Alice"])  // 30
```

Тип записывается как `map[KeyType]ValueType`:

- `map[string]int` — ключи-строки, значения-int
- `map[int]string` — ключи-int, значения-строки
- `map[string][]int` — значения это срезы

---

### 14.2 Создание карты

```go
// Литерал с данными
m1 := map[string]int{"a": 1, "b": 2}

// Пустая карта через make
m2 := make(map[string]int)

// Пустой литерал
m3 := map[string]int{}

// nil-карта (zero value) — ОПАСНО для записи!
var m4 map[string]int
fmt.Println(m4 == nil)  // true
```

> ⚠️ **Критично:** в `nil`-карту нельзя писать — это паника! Читать из неё можно (вернёт zero value), но `m4["key"] = 1` вызовет `panic: assignment to entry in nil map`. Всегда инициализируй карту через `make` или литерал перед записью.

---

### 14.3 Операции с картой

```go
m := make(map[string]int)

// Запись
m["apple"] = 5
m["banana"] = 3

// Чтение
fmt.Println(m["apple"])   // 5

// Чтение несуществующего ключа → zero value (НЕ ошибка!)
fmt.Println(m["missing"]) // 0 (zero value для int)

// Обновление
m["apple"] = 10

// Удаление
delete(m, "banana")

// Размер
fmt.Println(len(m))       // количество пар
```

---

### 14.4 Проверка наличия ключа — паттерн "comma ok"

Поскольку чтение несуществующего ключа возвращает zero value, нельзя отличить «ключа нет» от «ключ есть со значением 0». Для этого — второе возвращаемое значение:

```go
m := map[string]int{"apple": 5, "zero": 0}

value, ok := m["apple"]
fmt.Println(value, ok)  // 5 true

value, ok = m["zero"]
fmt.Println(value, ok)  // 0 true — ключ ЕСТЬ, значение 0

value, ok = m["missing"]
fmt.Println(value, ok)  // 0 false — ключа НЕТ
```

> 💡 Это **ключевой паттерн** работы с картами. `ok` — булев флаг наличия. Ты уже встречал его в мемоизации (урок 10): `v, ok := cache[n]`.

Часто используется в `if` с инициализацией:

```go
if value, ok := m["key"]; ok {
    fmt.Println("найдено:", value)
} else {
    fmt.Println("не найдено")
}
```

### 14.5 Итерация по карте

```go
m := map[string]int{"a": 1, "b": 2, "c": 3}

// Ключ и значение
for key, value := range m {
    fmt.Printf("%s = %d\n", key, value)
}

// Только ключи
for key := range m {
    fmt.Println(key)
}

// Только значения (ключ игнорируем)
for _, value := range m {
    fmt.Println(value)
}
```

> ⚠️ **ВАЖНО:** порядок итерации по карте **случайный** и меняется от запуска к запуску! Go намеренно рандомизирует его, чтобы код не полагался на порядок. Если нужен определённый порядок — собери ключи в срез и отсортируй (см. пример 3).

---

### 14.6 Карта — ссылочный тип

Как и срез, карта — ссылочный тип. Передаётся в функцию как «ссылка» на одни данные:

```go
func addEntry(m map[string]int) {
    m["new"] = 100   // изменение ВИДНО снаружи
}

func main() {
    data := map[string]int{"a": 1}
    addEntry(data)
    fmt.Println(data)  // map[a:1 new:100] — изменилось!
}
```

> 💡 В отличие от срезов, тут нет нюанса с `append` — карта не «переселяется». Любое изменение карты в функции видно снаружи. Но `nil`-карту функция не сможет инициализировать для вызывающего (нужно вернуть новую).

---

### 14.7 Какие типы можно использовать как ключи

Ключом может быть только **сравнимый** тип (тот, что работает с `==`):

- ✅ числа, строки, bool, указатели, массивы из сравнимых типов, структуры из сравнимых полей
- ❌ срезы, карты, функции — **нельзя** (они не сравнимы через `==`)

```go
m1 := map[string]int{}   // ✅
m2 := map[int]string{}   // ✅
m3 := map[[2]int]string{} // ✅ массив сравним
// m4 := map[[]int]string{} // ❌ срез не сравним — ошибка компиляции
```

> 💡 Помнишь из урока 12-13: массивы сравнимы (`==`), срезы нет. Поэтому массив можно использовать ключом, а срез — нельзя.

---

### 14.8 Карта значений-срезов (частый паттерн)

```go
// Группировка: слово → список его синонимов
synonyms := map[string][]string{
    "happy": {"glad", "joyful"},
    "sad":   {"unhappy", "down"},
}

synonyms["happy"] = append(synonyms["happy"], "cheerful")
fmt.Println(synonyms["happy"])  // [glad joyful cheerful]
```

> 💡 `append(m[key], value)` работает даже если ключа ещё нет: чтение `m[key]` вернёт `nil`-срез, а `append` к nil-срезу создаёт новый. Удобно для группировки.

---

### 14.9 Множество (set) через карту

В Go нет встроенного типа «множество», но его делают через `map[T]bool` или `map[T]struct{}`:

```go
// Множество через map[T]bool
seen := map[string]bool{}
seen["apple"] = true

if seen["apple"] {
    fmt.Println("уже видели apple")
}

// Через map[T]struct{} — экономит память (struct{} занимает 0 байт)
set := map[string]struct{}{}
set["banana"] = struct{}{}
_, exists := set["banana"]
fmt.Println(exists)  // true
```

> 💡 `map[T]struct{}` — идиома для множества когда важна только память: пустая структура `struct{}` не занимает места, в отличие от `bool` (1 байт). Для большинства случаев `map[T]bool` читается проще.

## 💡 Примеры

### Пример 1 — Подсчёт частоты

```go
package main

import "fmt"

func main() {
    text := []string{"a", "b", "a", "c", "b", "a"}
    counts := make(map[string]int)

    for _, word := range text {
        counts[word]++   // если ключа нет — начнёт с 0, потом +1
    }

    fmt.Println(counts)  // map[a:3 b:2 c:1]
}
```

### Пример 2 — comma ok в действии

```go
package main

import "fmt"

func main() {
    inventory := map[string]int{
        "sword":  1,
        "shield": 0,  // есть в инвентаре, но 0 штук
    }

    check := func(item string) {
        if count, ok := inventory[item]; ok {
            fmt.Printf("%s: %d шт\n", item, count)
        } else {
            fmt.Printf("%s: нет в инвентаре\n", item)
        }
    }

    check("sword")   // sword: 1 шт
    check("shield")  // shield: 0 шт  ← есть, но ноль
    check("potion")  // potion: нет в инвентаре
}
```

### Пример 3 — Сортированная итерация

```go
package main

import (
    "fmt"
    "sort"
)

func main() {
    m := map[string]int{"banana": 3, "apple": 5, "cherry": 1}

    // Собираем ключи в срез
    keys := make([]string, 0, len(m))
    for k := range m {
        keys = append(keys, k)
    }

    // Сортируем
    sort.Strings(keys)

    // Итерируем в порядке
    for _, k := range keys {
        fmt.Printf("%s: %d\n", k, m[k])
    }
    // apple: 5
    // banana: 3
    // cherry: 1
}
```

### Пример 4 — Множество уникальных

```go
package main

import "fmt"

func unique(nums []int) []int {
    seen := make(map[int]bool)
    result := []int{}

    for _, n := range nums {
        if !seen[n] {
            seen[n] = true
            result = append(result, n)
        }
    }
    return result
}

func main() {
    nums := []int{1, 2, 2, 3, 3, 3, 4}
    fmt.Println(unique(nums))  // [1 2 3 4]
}
```

## ⚠️ Подводные камни

### 1. Запись в nil-карту — паника

```go
var m map[string]int
m["key"] = 1  // panic: assignment to entry in nil map
// Решение: m := make(map[string]int)
```

### 2. Чтение несуществующего ключа не ошибка

```go
m := map[string]int{}
fmt.Println(m["missing"])  // 0, не паника — это zero value
// Чтобы отличить "нет ключа" — используй comma ok
```

### 3. Порядок итерации случайный

```go
m := map[string]int{"a": 1, "b": 2, "c": 3}
for k := range m {
    // порядок РАЗНЫЙ при каждом запуске!
}
// Нужен порядок — собери ключи и отсортируй
```

### 4. Срез нельзя использовать как ключ

```go
// m := map[[]int]string{}  // ❌ ошибка компиляции
```

### 5. Нельзя взять адрес элемента карты

```go
m := map[string]int{"a": 1}
// p := &m["a"]  // ❌ ошибка: нельзя взять адрес значения в карте
// Причина: карта может переразместить данные при росте
```

### 6. Карты нельзя сравнивать через ==

```go
m1 := map[string]int{"a": 1}
m2 := map[string]int{"a": 1}
// m1 == m2  // ❌ только m1 == nil
// Для сравнения: maps.Equal(m1, m2) из пакета maps
```

## 📋 Краткое резюме

- Карта — структура «ключ-значение» (хеш-таблица); тип `map[K]V`.
- Создание: литерал, `make(map[K]V)`; zero value = `nil`.
- **В nil-карту нельзя писать** — паника; читать можно (вернёт zero value). Всегда инициализируй через `make`.
- Чтение несуществующего ключа возвращает zero value, не ошибку.
- Паттерн **comma ok** (`v, ok := m[k]`) отличает «ключа нет» от «значение = zero».
- Операции: запись `m[k]=v`, удаление `delete(m, k)`, размер `len(m)`.
- **Порядок итерации случайный** — для упорядоченного обхода собери ключи в срез и отсортируй.
- Карта — ссылочный тип: изменения в функции видны снаружи.
- Ключ — только сравнимый тип (числа, строки, массивы); срезы/карты/функции ключами быть не могут.
- Множество (set) делают через `map[T]bool` или `map[T]struct{}`.
- Карты нельзя сравнивать через `==` (только с nil); адрес элемента карты взять нельзя.
