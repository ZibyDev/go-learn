## 📌 Теория

### 6.1 if — базовый синтаксис

```go
if x > 10 {
    fmt.Println("больше 10")
} else if x > 5 {
    fmt.Println("больше 5")
} else {
    fmt.Println("5 или меньше")
}
```

Отличия от того, что ты знаешь:

1. **Скобки `()` вокруг условия не нужны** (и считаются неидиоматичными)
2. **Фигурные скобки `{}` обязательны** — даже для одной строки
3. **Открывающая `{` — на той же строке** (иначе ошибка компиляции)
4. **Нет тернарного оператора** — никакого `x > 0 ? a : b`

```go
// ❌ Так нельзя — нет тернарника в Go
result := x > 0 ? "положительное" : "отрицательное"

// ✅ Только через if
var result string
if x > 0 {
    result = "положительное"
} else {
    result = "отрицательное"
}
```

> 💡 Отсутствие тернарного оператора — намеренное решение. Авторы Go считают, что обычный `if` читается яснее. Поначалу непривычно после JS/C#, но привыкаешь.

### 6.2 if с инициализацией — важная фишка

В Go можно объявить переменную прямо в `if`. Она будет видна только внутри блока `if/else`:

```go
if x := compute(); x > 10 {
    fmt.Println("большое значение:", x)
} else {
    fmt.Println("маленькое:", x)  // x доступна и здесь
}
// здесь x уже НЕ доступна
```

Это **самый частый паттерн** в Go — особенно с обработкой ошибок:

```go
if err := doSomething(); err != nil {
    fmt.Println("ошибка:", err)
    return
}
// дальше код, где err уже не нужна и не засоряет область видимости
```

> 💡 Это решает проблему "протекания" переменных. `err` живёт только там, где нужна. Запомни этот паттерн — он встречается в Go коде постоянно.

> ⚠️ Помни про shadowing из урока 2! `x := compute()` в `if` создаёт новую переменную, даже если `x` уже есть снаружи.

### 6.3 Условие — только bool

В Go условие должно быть строго типа `bool`. Никаких "истинных" чисел или строк как в Python/JS:

```go
x := 5
// if x { }       ❌ ошибка: x это int, не bool
if x != 0 { }     // ✅

s := ""
// if s { }       ❌
if s != "" { }    // ✅

if len(s) > 0 { } // ✅
```

> 💡 Это убирает целый класс багов с неявным приведением. `if 0`, `if ""`, `if nil` — всё это ошибки компиляции, не молчаливое поведение.

### 6.4 switch — базовый

`switch` в Go мощнее, чем в большинстве языков:

```go
switch day {
case "Monday":
    fmt.Println("понедельник")
case "Saturday", "Sunday":   // несколько значений в одном case
    fmt.Println("выходной")
default:
    fmt.Println("обычный день")
}
```

Ключевые отличия от C/Java/C#:

1. **Нет `break`** — каждый `case` автоматически прерывается (нет "проваливания")
2. **Можно несколько значений** в одном `case` через запятую
3. **`default` может быть в любом месте** (но обычно в конце)

> 💡 В C/Java забытый `break` — частый баг (проваливание в следующий case). В Go наоборот — провал не происходит по умолчанию. Если он нужен — есть `fallthrough`.

---

### 6.5 switch с fallthrough

Если всё-таки нужно "провалиться" в следующий case (как в C по умолчанию):

```go
switch x {
case 1:
    fmt.Println("один")
    fallthrough        // принудительно проваливаемся в следующий case
case 2:
    fmt.Println("два")  // выполнится тоже, если x == 1
default:
    fmt.Println("другое")
}
// При x == 1 выведет: "один" и "два"
```

> ⚠️ `fallthrough` используется редко. Если видишь его в коде — обычно это спорное решение. Но знать надо.

---

### 6.6 switch без условия (замена if-else цепочек)

`switch` без выражения = элегантная замена длинной цепочки `if-else if`:

```go
// Вместо:
if score >= 90 {
    grade = "A"
} else if score >= 80 {
    grade = "B"
} else if score >= 70 {
    grade = "C"
} else {
    grade = "F"
}

// Можно:
switch {
case score >= 90:
    grade = "A"
case score >= 80:
    grade = "B"
case score >= 70:
    grade = "C"
default:
    grade = "F"
}
```

> 💡 `switch {` без выражения — каждый case это булево условие. Часто читается чище, чем `if-else if`.

---

### 6.7 switch с инициализацией

Как и `if`, `switch` поддерживает инициализацию:

```go
switch x := getValue(); {
case x < 0:
    fmt.Println("отрицательное")
case x == 0:
    fmt.Println("ноль")
default:
    fmt.Println("положительное")
}
```

Или с выражением:

```go
switch day := getDay(); day {
case "Sat", "Sun":
    fmt.Println("выходной")
default:
    fmt.Println("будни")
}
```

---

### 6.8 type switch (забегаем вперёд)

Существует особый `switch` по типу значения. Подробно разберём в уроке 18 (пустой интерфейс), здесь только упомяну что он есть:

```go
switch v := value.(type) {
case int:
    fmt.Println("это int:", v)
case string:
    fmt.Println("это string:", v)
}
```

> 💡 Не пытайся это пока использовать — нужен материал про интерфейсы. Просто знай что `switch` умеет и так.

## 💡 Примеры

### Пример 1 — if с инициализацией и обработкой ошибки

```go
package main

import (
    "fmt"
    "strconv"
)

func main() {
    input := "123"

    // strconv.Atoi возвращает (int, error)
    if num, err := strconv.Atoi(input); err != nil {
        fmt.Println("не число:", err)
    } else {
        fmt.Println("число + 10 =", num+10)
    }
}
```

### Пример 2 — switch для классификации

```go
package main

import "fmt"

func classify(n int) string {
    switch {
    case n < 0:
        return "отрицательное"
    case n == 0:
        return "ноль"
    case n < 10:
        return "однозначное"
    case n < 100:
        return "двузначное"
    default:
        return "большое"
    }
}

func main() {
    fmt.Println(classify(-5))   // отрицательное
    fmt.Println(classify(0))    // ноль
    fmt.Println(classify(7))    // однозначное
    fmt.Println(classify(42))   // двузначное
    fmt.Println(classify(1000)) // большое
}
```

### Пример 3 — switch с несколькими значениями

```go
package main

import "fmt"

func dayType(day string) string {
    switch day {
    case "Sat", "Sun":
        return "выходной"
    case "Mon", "Tue", "Wed", "Thu", "Fri":
        return "рабочий"
    default:
        return "неизвестный день"
    }
}

func main() {
    fmt.Println(dayType("Sat"))  // выходной
    fmt.Println(dayType("Mon"))  // рабочий
    fmt.Println(dayType("xyz"))  // неизвестный день
}
```

### Пример 4 — Игровая логика урона (gamedev)

```go
package main

import "fmt"

func damageMultiplier(weaponType string) float64 {
    switch weaponType {
    case "sword", "axe":
        return 1.5
    case "bow":
        return 1.2
    case "staff":
        return 2.0
    default:
        return 1.0
    }
}

func main() {
    baseDamage := 10.0
    weapon := "staff"

    total := baseDamage * damageMultiplier(weapon)
    fmt.Printf("Оружие %s наносит %.1f урона\n", weapon, total)
    // Оружие staff наносит 20.0 урона
}
```

## ⚠️ Подводные камни

### 1. Условие — только bool

```go
if 1 { }        // ❌ non-bool used as if condition
if x { }        // ❌ если x не bool
```

### 2. Нет тернарного оператора

```go
x := a > b ? a : b  // ❌ не существует в Go
```

### 3. Переменная из if не видна снаружи

```go
if x := 10; x > 5 {
    // x видна
}
// fmt.Println(x)  ❌ x вне области видимости
```

### 4. switch не проваливается (в отличие от C)

```go
switch x {
case 1:
    doA()  // выполнится только это
case 2:
    doB()  // НЕ выполнится при x==1 (нет fallthrough)
}
```

### 5. Открывающая скобка на той же строке

```go
if x > 0
{               // ❌ ошибка компиляции
}
```

## 📋 Краткое резюме

- `if` без скобок вокруг условия, но фигурные `{}` обязательны; открывающая `{` на той же строке.
- Тернарного оператора нет — только полный `if/else`.
- `if x := f(); cond` — инициализация в условии, переменная видна только в блоке; ключевой паттерн с ошибками.
- Условие строго `bool` — никаких `if 0` / `if ""`.
- `switch` не проваливается по умолчанию (нет авто-`break`); провал — через `fallthrough`.
- Один `case` может содержать несколько значений через запятую.
- `switch {}` без выражения — элегантная замена цепочки `if-else if`.
- `switch` выбирает **одну** ветку; для независимых проверок нужна серия отдельных `if`.
