## 📌 Теория

### 1.1 Что такое Go

Go (или Golang) — компилируемый, статически типизированный язык, созданный в Google в 2009 году. Его главные особенности:

- **Простой синтаксис** — язык намеренно маленький, без лишних конструкций
- **Быстрая компиляция** — компилируется в нативный бинарник за секунды
- **Встроенная конкурентность** — горутины и каналы из коробки
- **Сборщик мусора** — управление памятью автоматическое, но предсказуемое
- **Один бинарник** — при сборке всё включается в один исполняемый файл

> 💡 **Для тебя:** в отличие от Python — компилируемый и статически типизированный. В отличие от C++ — нет ручного управления памятью и намного проще синтаксис. В отличие от C# и Kotlin — нет классов и наследования в привычном смысле.

### 1.2 Установка

Идём на [https://go.dev/dl/](https://go.dev/dl/) и скачиваем актуальную версию под свою ОС.

---

#### 🍎 macOS

**Способ 1 — через Homebrew (рекомендуется):**

```bash
# Установить Homebrew если ещё нет
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Установить Go
brew install go

# Проверка
go version
# go version go1.22.0 darwin/arm64
```

**Способ 2 — через pkg установщик:** Скачать `.pkg` файл с [go.dev/dl](https://go.dev/dl/), запустить и следовать инструкциям. Go установится в `/usr/local/go`.

**Настройка PATH для macOS** — добавить в `~/.zshrc` (zsh — оболочка по умолчанию на Mac):

```bash
export GOPATH=$HOME/go
export PATH=$PATH:$GOPATH/bin
```

Применить:

```bash
source ~/.zshrc
```

---

#### 🐧 Ubuntu (Linux)

```bash
# Удаляем старую версию если есть
sudo rm -rf /usr/local/go

# Скачиваем (замени версию на актуальную с go.dev/dl)
wget https://go.dev/dl/go1.22.0.linux-amd64.tar.gz

# Распаковываем
sudo tar -C /usr/local -xzf go1.22.0.linux-amd64.tar.gz

# Добавляем в PATH — вставить в ~/.bashrc или ~/.zshrc
export PATH=$PATH:/usr/local/go/bin
export GOPATH=$HOME/go
export PATH=$PATH:$GOPATH/bin

# Применяем
source ~/.bashrc
```

> 💡 **Apple Silicon (M1/M2/M3):** при установке через Homebrew всё работает нативно для arm64. Если скачиваешь вручную — выбирай `darwin/arm64`, а не `darwin/amd64`.

---

**Проверка (на обеих системах):**

```bash
go version
go env GOPATH
go env GOROOT
```

### 1.3 Структура рабочего окружения

```
$HOME/
└── go/                  ← GOPATH (создаётся автоматически)
    ├── bin/             ← сюда устанавливаются go install инструменты
    ├── pkg/             ← кэш скомпилированных пакетов
    └── src/             ← устаревший способ, сейчас не используется
```

> ⚠️ **Важно:** начиная с Go 1.11 появились **модули** (`go mod`), и больше не нужно держать проекты внутри `GOPATH/src`. Проект можно создавать в любом месте файловой системы.

---

### 1.4 Создание проекта — Go модули

Модуль — это единица распространения кода в Go. Каждый проект начинается с инициализации модуля:

```bash
# Создаём папку проекта
mkdir hello-go
cd hello-go

# Инициализируем модуль
go mod init hello-go
```

После этого создаётся файл `go.mod`:

```
module hello-go

go 1.22.0
```

- `module hello-go` — имя модуля (обычно путь к репозиторию, например `github.com/username/project`)
- `go 1.22.0` — минимальная версия Go

---

### 1.5 Структура Go проекта

**Минимальный проект:**

```
hello-go/
├── go.mod
└── main.go
```

**Типичный небольшой проект:**

```
myproject/
├── go.mod
├── go.sum          ← хэши зависимостей (генерируется автоматически)
├── main.go         ← точка входа
├── internal/       ← внутренние пакеты (не доступны снаружи модуля)
│   └── utils/
│       └── utils.go
└── pkg/            ← публичные пакеты (можно импортировать извне)
    └── models/
        └── models.go
```

> 💡 `internal/` — это соглашение, которое Go **принудительно** соблюдает: пакеты внутри `internal` нельзя импортировать из других модулей. Это не просто конвенция, это ограничение компилятора.

### 1.6 Первая программа

Создаём файл `main.go`:

```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, Go!")
}
```

Разбираем по строкам:

|Строка|Что означает|
|---|---|
|`package main`|Этот файл принадлежит пакету `main` — точке входа программы|
|`import "fmt"`|Импортируем стандартный пакет для форматированного вывода|
|`func main()`|Главная функция — с неё начинается выполнение программы|
|`fmt.Println(...)`|Печатаем строку с переносом строки в конце|

> ⚠️ **Важно:** в Go **нет точки с запятой** в конце строк. Компилятор добавляет их автоматически по правилу: если строка заканчивается на идентификатор, литерал, `)`, `]`, `}` — добавляется `;`. Это объясняет почему открывающая `{` ВСЕГДА на той же строке — иначе компилятор поставит `;` перед ней и получит ошибку.

### 1.7 Основные команды go tool

```bash
# Запустить программу без компиляции в файл
go run main.go

# Скомпилировать в бинарник
go build -o myapp main.go

# Скомпилировать и установить в $GOPATH/bin
go install

# Добавить зависимость
go get github.com/some/package

# Подтянуть все зависимости из go.mod
go mod tidy

# Форматировать код (ВСЕГДА делать перед коммитом)
go fmt ./...

# Проверить код на типичные ошибки
go vet ./...

# Запустить тесты
go test ./...
```

> 💡 `./...` — это паттерн означающий "текущий пакет и все вложенные". Используется почти везде в go командах.

### 1.9 Как Go отличается от того, что ты знаешь

|Концепция|Python/JS/C#|Go|
|---|---|---|
|Типизация|Динамическая / статическая|Строго статическая|
|Классы|Есть|Нет (структуры + методы)|
|Наследование|Есть|Нет (только композиция)|
|Исключения|try/catch|Нет (error как возвращаемое значение)|
|Null|None/null/nil|nil (только для указателей, срезов, карт и т.д.)|
|Точка с запятой|Обычно есть|Нет|
|Неиспользуемые переменные|Предупреждение|**Ошибка компиляции**|
|Неиспользуемые импорты|Предупреждение|**Ошибка компиляции**|
|Форматирование кода|Опционально|`gofmt` — единый стандарт|

> ⚠️ **Главный шок для новичков из других языков:** в Go неиспользуемая переменная или импорт — это **ошибка компиляции**, не предупреждение. Программа просто не соберётся.

## 💡 Примеры

### Пример 1 — Несколько пакетов в импорте

```go
package main

import (
    "fmt"
    "math"
    "os"
)

func main() {
    fmt.Println("Квадратный корень из 16:", math.Sqrt(16))
    fmt.Println("Имя программы:", os.Args[0])
}
```

### Пример 2 — Алиасы импортов

```go
package main

import (
    "fmt"
    m "math"        // алиас m для пакета math
    _ "os"          // импорт только ради побочных эффектов (init функции)
)

func main() {
    fmt.Println(m.Pi)
}
```

### Пример 3 — Несколько файлов в одном пакете

```
myproject/
├── go.mod
├── main.go
└── greet.go
```

`greet.go`:

```go
package main

import "fmt"

func greet(name string) {
    fmt.Println("Привет,", name)
}
```

`main.go`:

```go
package main

func main() {
    greet("Go")  // функция из greet.go доступна напрямую
}
```

> 💡 Все файлы с одинаковым `package` в одной папке — это один пакет. Не нужно явно импортировать соседние файлы.

## ⚠️ Подводные камни

1. **Открывающая скобка — только на той же строке:**

```go
// ✅ Правильно
func main() {
}

// ❌ Ошибка компиляции
func main()
{
}
```

2. **Неиспользуемый импорт — ошибка:**

```go
import "fmt"
import "os"  // если os нигде не используется — программа не скомпилируется
```

3. **`go run` vs `go build`:** `go run` каждый раз компилирует заново во временную папку. Для разработки — удобно. Для продакшна — используй `go build`.
    
4. **Имя модуля важно:** если планируешь публиковать пакет, имя модуля должно совпадать с путём репозитория: `github.com/username/myproject`

## 📋 Краткое резюме

- Go — компилируемый статически типизированный язык; результат сборки — один бинарник.
- Установка: macOS через Homebrew (`brew install go`), Ubuntu через `tar` в `/usr/local`.
- Любой проект начинается с модуля: `go mod init <имя>` создаёт `go.mod`.
- Открывающая `{` всегда на той же строке — иначе ошибка компиляции.
- Неиспользуемые переменные и импорты — **ошибка компиляции**, не предупреждение.
- Основные команды: `go run`, `go build`, `go fmt`, `go vet`, `go test`; `./...` — текущий пакет и вложенные.
- `internal/` — пакеты, недоступные извне модуля (ограничение компилятора, не соглашение).
