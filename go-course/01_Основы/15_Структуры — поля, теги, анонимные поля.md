## 📌 Теория

### 15.1 Что такое структура

Структура (struct) — это составной тип, объединяющий несколько именованных полей. Это способ создать свой тип данных из других типов. Аналог `class` (без методов) в других языках, но в Go классов нет — структуры с методами их заменяют (методы будут в уроке 16).

```go
type Player struct {
    Name   string
    Health int
    Level  int
}

p := Player{Name: "Hero", Health: 100, Level: 1}
fmt.Println(p.Name)  // Hero
```

> 💡 Структура — фундамент моделирования данных в Go. Вместо «классов с наследованием» Go предлагает «структуры с композицией» (это урок 19-20).

---

### 15.2 Объявление и создание

```go
type Point struct {
    X, Y int   // поля одного типа можно в одну строку
}

// Способ 1 — именованные поля (рекомендуется)
p1 := Point{X: 1, Y: 2}

// Способ 2 — позиционный (порядок важен, хрупко)
p2 := Point{3, 4}

// Способ 3 — нулевая структура, потом заполнить
var p3 Point          // {0 0} — все поля zero value
p3.X = 5

// Способ 4 — указатель на структуру через &
p4 := &Point{X: 1, Y: 2}
```

> ⚠️ Позиционный способ `Point{3, 4}` хрупок: если добавишь поле или поменяешь порядок — код сломается молча. Всегда предпочитай именованные поля `Point{X: 3, Y: 4}`.

---

### 15.3 Доступ к полям

```go
type Player struct {
    Name   string
    Health int
}

p := Player{Name: "Hero", Health: 100}

// Чтение
fmt.Println(p.Name)    // Hero

// Запись
p.Health = 80

// Через указатель — Go автоматически разыменовывает
ptr := &p
fmt.Println(ptr.Name)  // Hero — не нужно (*ptr).Name
ptr.Health = 50        // тоже работает напрямую
```

> 💡 Как с указателями из урока 11: `ptr.Field` для указателя на структуру автоматически означает `(*ptr).Field`. Точка работает и для значения, и для указателя.

### 15.4 Zero value структуры

Структура без инициализации — все поля получают свои zero values:

```go
type Config struct {
    Host    string
    Port    int
    Debug   bool
}

var c Config
fmt.Printf("%+v\n", c)  // {Host: Port:0 Debug:false}
```

> 💡 `%+v` в `Printf` печатает структуру с именами полей — очень удобно для отладки. Просто `%v` даст `{ 0 false}` без имён.

---

### 15.5 Структуры — значимый тип (копируются)

Как и массивы (урок 12), структуры **копируются** при присваивании и передаче в функцию:

```go
type Point struct{ X, Y int }

a := Point{1, 2}
b := a          // ПОЛНАЯ копия
b.X = 99
fmt.Println(a)  // {1 2} — оригинал не тронут
fmt.Println(b)  // {99 2}
```

То же в функциях:

```go
func modify(p Point) {
    p.X = 999   // меняется копия
}

func modifyPtr(p *Point) {
    p.X = 999   // меняется оригинал через указатель
}
```

> ⚠️ Для больших структур копирование дорого. И если нужно изменить оригинал — передавай указатель `*Point`. Это та же логика, что с массивами и из урока 11.

### 15.6 Сравнение структур

Структуры сравнимы через `==`, если все их поля сравнимы (поэлементно):

```go
type Point struct{ X, Y int }

a := Point{1, 2}
b := Point{1, 2}
c := Point{3, 4}

fmt.Println(a == b)  // true
fmt.Println(a == c)  // false
```

> ⚠️ Если в структуре есть несравнимое поле (срез, карта, функция) — структуру нельзя сравнить через `==`:

```go
type Bad struct {
    Items []int  // срез несравним
}
// Bad{} == Bad{}  // ❌ ошибка компиляции
```

Сравнимые структуры можно использовать как **ключи карты**:

```go
visited := map[Point]bool{}
visited[Point{1, 2}] = true   // ✅ Point сравним
```

### 15.7 Вложенные структуры

Структуры могут содержать другие структуры как поля:

```go
type Address struct {
    City   string
    Street string
}

type Person struct {
    Name    string
    Address Address   // вложенная структура
}

p := Person{
    Name: "Alice",
    Address: Address{
        City:   "Moscow",
        Street: "Lenina 1",
    },
}

fmt.Println(p.Address.City)  // Moscow — доступ через цепочку
```

### 15.8 Анонимные поля (embedding) — встраивание

Можно встроить тип в структуру **без имени поля** — только тип. Это называется встраивание (embedding) и это основа композиции в Go:

```go
type Animal struct {
    Name string
}

type Dog struct {
    Animal   // анонимное поле — встроили Animal
    Breed string
}

d := Dog{
    Animal: Animal{Name: "Rex"},
    Breed:  "Labrador",
}

// Поля встроенного типа доступны НАПРЯМУЮ (промоушен полей)
fmt.Println(d.Name)         // Rex — не нужно d.Animal.Name!
fmt.Println(d.Animal.Name)  // Rex — но и так тоже можно
```

> 💡 Встраивание «поднимает» поля встроенного типа на верхний уровень. Это НЕ наследование — это композиция. Подробно разберём в уроке 19, здесь — основы.

### 15.9 Теги структур

Поля могут иметь **теги** — строковые метаданные, которые читаются через рефлексию. Чаще всего для JSON:

```go
type User struct {
    Name  string `json:"name"`
    Email string `json:"email"`
    Age   int    `json:"age,omitempty"`
}
```

Тег `json:"name"` говорит JSON-кодировщику использовать `name` вместо `Name`. `omitempty` — пропустить поле, если оно пустое.

> 💡 Теги сами по себе ничего не делают — это просто строки. Их интерпретируют пакеты (`encoding/json`, ORM, валидаторы). Подробно JSON и теги — в уроке 43. Здесь — что они есть и как выглядят.

### 15.10 Анонимные структуры

Можно создать структуру без объявления именованного типа — на месте:

```go
// Структура без имени типа — для разовых случаев
config := struct {
    Host string
    Port int
}{
    Host: "localhost",
    Port: 8080,
}

fmt.Println(config.Host)  // localhost
```

> 💡 Используется для одноразовых данных: временная группировка, табличные тесты (урок 28), разбор JSON «на лету». Если структура нужна в нескольких местах — объявляй именованный тип.

### 15.11 Конструкторы — идиома New

В Go нет конструкторов как в ООП-языках. Вместо них — функции-фабрики по соглашению `NewТип`:

```go
type Server struct {
    host string
    port int
}

// Конструктор по соглашению — возвращает указатель
func NewServer(host string, port int) *Server {
    return &Server{
        host: host,
        port: port,
    }
}

func main() {
    s := NewServer("localhost", 8080)
    fmt.Println(s.host)  // localhost
}
```

> 💡 Это просто соглашение, не синтаксис языка. `NewX` обычно возвращает `*X`. Полезно для валидации полей или установки значений по умолчанию при создании.

## 💡 Примеры

### Пример 1 — Создание и доступ

```go
package main

import "fmt"

type Rectangle struct {
    Width  float64
    Height float64
}

func main() {
    r := Rectangle{Width: 3, Height: 4}

    area := r.Width * r.Height
    fmt.Printf("%+v, площадь = %.1f\n", r, area)
    // {Width:3 Height:4}, площадь = 12.0
}
```

### Пример 2 — Значение vs указатель в функциях

```go
package main

import "fmt"

type Counter struct {
    Value int
}

func incValue(c Counter) {
    c.Value++  // копия
}

func incPointer(c *Counter) {
    c.Value++  // оригинал
}

func main() {
    c := Counter{Value: 0}

    incValue(c)
    fmt.Println(c.Value)  // 0 — не изменилось

    incPointer(&c)
    fmt.Println(c.Value)  // 1 — изменилось
}
```

### Пример 3 — Структура как ключ карты

```go
package main

import "fmt"

type Coord struct {
    X, Y int
}

func main() {
    // Карта посещённых клеток (частый паттерн в gamedev/алгоритмах)
    visited := map[Coord]bool{}

    visited[Coord{0, 0}] = true
    visited[Coord{1, 2}] = true

    fmt.Println(visited[Coord{0, 0}])  // true
    fmt.Println(visited[Coord{5, 5}])  // false — не посещали
}
```

### Пример 4 — Встраивание

```go
package main

import "fmt"

type Entity struct {
    X, Y int
}

type Enemy struct {
    Entity      // встроено
    Health int
}

func main() {
    e := Enemy{
        Entity: Entity{X: 10, Y: 20},
        Health: 100,
    }

    // Поля Entity доступны напрямую
    fmt.Println(e.X, e.Y, e.Health)  // 10 20 100
    e.X = 15  // меняем встроенное поле напрямую
    fmt.Println(e.X)  // 15
}
```

### Пример 5 — Конструктор New

```go
package main

import "fmt"

type Account struct {
    owner   string
    balance int
}

func NewAccount(owner string) *Account {
    return &Account{
        owner:   owner,
        balance: 0,  // начальный баланс
    }
}

func main() {
    acc := NewAccount("Alice")
    fmt.Printf("%+v\n", *acc)  // {owner:Alice balance:0}
}
```

## ⚠️ Подводные камни

### 1. Позиционная инициализация хрупка

```go
type P struct{ X, Y, Z int }
p := P{1, 2, 3}  // сломается если добавить/переставить поля
// Лучше: P{X: 1, Y: 2, Z: 3}
```

### 2. Структура копируется при передаче

```go
func f(p Point) { p.X = 9 }  // меняет копию
// чтобы менять оригинал — *Point
```

### 3. Структура со срезом/картой несравнима

```go
type S struct{ items []int }
// S{} == S{}  // ❌ ошибка компиляции
```

### 4. Копия структуры со срезом делит данные среза

```go
type Box struct{ items []int }
a := Box{items: []int{1, 2, 3}}
b := a            // копируется заголовок среза, но данные ОБЩИЕ!
b.items[0] = 99
fmt.Println(a.items)  // [99 2 3] — изменилось! (срез ссылочный)
```

> ⚠️ Это важная тонкость: копирование структуры копирует поля **поверхностно**. Поля-срезы/карты после копии всё ещё ссылаются на те же данные. Для глубокой копии — копируй срезы вручную.

### 5. Нельзя взять адрес поля в карте

```go
m := map[string]Point{"a": {1, 2}}
// m["a"].X = 5  // ❌ нельзя менять поле структуры в карте напрямую
// Нужно: достать, изменить, положить обратно
p := m["a"]
p.X = 5
m["a"] = p
```

## 📋 Краткое резюме

- Структура — составной тип из именованных полей; в Go заменяет классы (методы — урок 16).
- Создание: именованные поля `T{X: 1}` (предпочтительно), позиционно `T{1, 2}` (хрупко), `var t T` (zero values).
- Доступ через точку; для указателя `ptr.Field` = `(*ptr).Field` автоматически.
- `%+v` печатает структуру с именами полей — удобно для отладки.
- Структура — значимый тип: **копируется** при присваивании и передаче; для изменения оригинала — указатель `*T`.
- Сравнима через `==`, если все поля сравнимы; тогда может быть ключом карты.
- Структура со срезом/картой несравнима (нельзя `==`, нельзя ключом).
- Вложенные структуры — поле-структура; встраивание (embedding) — анонимное поле, поля «поднимаются» наверх (промоушен).
- Теги (`json:"name"`) — метаданные для пакетов (JSON, ORM); сами по себе ничего не делают.
- Конструкторов нет — идиома `NewX`, возвращающая `*X`.
- **Поверхностное копирование:** копия структуры делит данные полей-срезов/карт с оригиналом.
