## 📌 Теория

### 5.1 Что такое строка в Go

Строка в Go — это **неизменяемая последовательность байт**. Не символов, а именно байт! Это ключевое отличие от многих языков и источник большинства "странностей".

```go
s := "Hello"
fmt.Println(len(s))  // 5 — длина в БАЙТАХ, не символах
```

Для ASCII-текста байт = символ, поэтому всё интуитивно. Но как только появляются не-латинские символы — начинается самое интересное.

```go
s := "Привет"
fmt.Println(len(s))  // 12, не 6! Каждая кириллическая буква занимает 2 байта
```

> 💡 Запомни сразу: `len(string)` возвращает количество **байт**, а не символов. Это фундамент, на котором строится понимание строк в Go.

---

### 5.2 UTF-8 — как Go хранит текст

Go хранит строки в кодировке **UTF-8**. Это переменная кодировка: один символ может занимать от 1 до 4 байт.

|Символы|Байт на символ|
|---|---|
|ASCII (a-z, A-Z, 0-9, знаки)|1 байт|
|Кириллица, латиница с диакритикой|2 байта|
|Большинство азиатских символов|3 байта|
|Эмодзи, редкие символы|4 байта|

```go
fmt.Println(len("a"))   // 1 (ASCII)
fmt.Println(len("я"))   // 2 (кириллица)
fmt.Println(len("中"))  // 3 (китайский)
fmt.Println(len("😀"))  // 4 (эмодзи)
```

> 💡 UTF-8 придумал Кен Томпсон — один из создателей Go. Поэтому язык изначально спроектирован вокруг UTF-8.

---

### 5.3 Байты vs руны

Здесь два ключевых типа:

- **`byte`** (псевдоним `uint8`) — один байт
- **`rune`** (псевдоним `int32`) — один Unicode code point (символ)

```go
s := "Яa"

// Индексация даёт БАЙТ, не символ!
fmt.Println(s[0])  // 208 — первый байт буквы Я, а не сама буква
fmt.Println(s[1])  // 175 — второй байт буквы Я
fmt.Println(s[2])  // 97  — буква a (ASCII)
```

> ⚠️ **Критически важно:** `s[i]` возвращает `byte`, а не символ! Для многобайтных символов это даст кусок, а не целую букву.

---

### 5.4 Доступ к символам — два способа итерации

#### Способ 1 — по байтам (обычный индекс)

```go
s := "Hello"
// этот цикл идёт по БАЙТАМ
i := 0
for i < len(s) {
    fmt.Printf("%d: %c\n", i, s[i])
    i++
}
```

Для ASCII работает. Для кириллицы — сломается, выдаст мусор.

#### Способ 2 — по рунам (range)

```go
s := "Привет"
// range по строке автоматически декодирует UTF-8 в руны
for index, r := range s {
    fmt.Printf("%d: %c (код %d)\n", index, r, r)
}
```

> 💡 `range` по строке — **умный**: он декодирует UTF-8 и выдаёт руны (целые символы), а `index` — это байтовая позиция начала символа. Это правильный способ перебирать символы.

```
0: П (код 1055)
2: р (код 1088)   ← индекс прыгнул на 2, т.к. П занял 2 байта
4: и (код 1080)
...
```

---

### 5.5 Подсчёт символов

```go
import "unicode/utf8"

s := "Привет"

fmt.Println(len(s))                    // 12 — байты
fmt.Println(utf8.RuneCountInString(s)) // 6 — символы (руны)
```

> 💡 Чтобы узнать количество **символов** (а не байт) — используй `utf8.RuneCountInString()`.

---

### 5.6 Преобразования: string ↔ byte ↔ rune

```go
s := "Привет"

// строка → срез байт
b := []byte(s)
fmt.Println(len(b))  // 12

// строка → срез рун
r := []rune(s)
fmt.Println(len(r))  // 6

// обратно
s2 := string(b)      // из байт
s3 := string(r)      // из рун
```

> 💡 `[]rune(s)` — удобный способ получить "массив символов" для работы по индексам. Но это создаёт копию, так что для больших строк дороговато.

> ⚠️ **Ловушка:** `string(65)` даст `"A"` (символ с кодом 65), а не `"65"`! Для числа в строку используй `strconv.Itoa()`.

```go
fmt.Println(string(65))        // "A" — НЕ "65"!
fmt.Println(strconv.Itoa(65))  // "65" — правильно
```

### 5.7 Обычные и сырые строки

```go
// Обычная строка — двойные кавычки, escape-последовательности работают
normal := "Строка\nс переносом\tи табом"

// Сырая строка — обратные кавычки, всё буквально
raw := `Строка\nбез переноса, \t буквально`

// Сырые строки удобны для путей, regex, многострочного текста
path := `C:\Users\name\file.txt`
multiline := `Первая строка
Вторая строка
Третья строка`
```

|Escape|Значение|
|---|---|
|`\n`|перенос строки|
|`\t`|табуляция|
|`\"`|двойная кавычка|
|`\\`|обратный слеш|
|`\r`|возврат каретки|
|`\u00E9`|Unicode символ (é)|

### 5.8 Конкатенация и форматирование

```go
// Простая конкатенация через +
first := "Hello"
second := "World"
result := first + " " + second  // "Hello World"

// Форматирование через fmt.Sprintf
name := "Hero"
level := 5
msg := fmt.Sprintf("%s достиг уровня %d", name, level)
// "Hero достиг уровня 5"
```

> ⚠️ Конкатенация через `+` в цикле — **неэффективна** (каждый раз создаётся новая строка, т.к. строки неизменяемы). Для построения строк в цикле используется `strings.Builder` — разберём в уроке 40.

### 5.9 Полезные глаголы fmt для строк

```go
s := "Go"
r := 'Я'

fmt.Printf("%s\n", s)   // Go         — строка
fmt.Printf("%q\n", s)   // "Go"       — в кавычках
fmt.Printf("%c\n", r)   // Я          — символ по коду руны
fmt.Printf("%d\n", r)   // 1071       — код руны как число
fmt.Printf("%U\n", r)   // U+042F     — Unicode позиция
fmt.Printf("%x\n", s)   // 476f       — hex представление байт
fmt.Printf("%v\n", s)   // Go         — значение по умолчанию
fmt.Printf("%T\n", s)   // string     — тип
```

## 💡 Примеры

### Пример 1 — len в байтах vs символах

```go
package main

import (
    "fmt"
    "unicode/utf8"
)

func main() {
    words := "Hello, Привет, 中文, 😀"

    fmt.Println("Байт:", len(words))
    fmt.Println("Символов:", utf8.RuneCountInString(words))
}
```

### Пример 2 — Правильный перебор символов

```go
package main

import "fmt"

func main() {
    s := "Gо"  // вторая буква — кириллическая 'о'!

    fmt.Println("--- По байтам (неправильно для не-ASCII) ---")
    i := 0
    for i < len(s) {
        fmt.Printf("байт %d: %d\n", i, s[i])
        i++
    }

    fmt.Println("--- По рунам через range (правильно) ---")
    for index, r := range s {
        fmt.Printf("позиция %d: %c (код %d)\n", index, r, r)
    }
}
```

### Пример 3 — Конвертация в руны и доступ по индексу

```go
package main

import "fmt"

func main() {
    s := "Привет"

    // Чтобы взять 3-й символ — конвертируем в []rune
    runes := []rune(s)
    fmt.Printf("3-й символ: %c\n", runes[2])  // и

    // Сравни — прямая индексация даст байт:
    fmt.Printf("3-й байт: %d\n", s[2])         // кусок буквы
}
```

### Пример 4 — string(int) ловушка

```go
package main

import (
    "fmt"
    "strconv"
)

func main() {
    num := 65

    // ❌ Ловушка
    wrong := string(rune(num))   // "A"
    fmt.Println("string(rune):", wrong)

    // ✅ Правильно
    correct := strconv.Itoa(num) // "65"
    fmt.Println("strconv.Itoa:", correct)
}
```

## ⚠️ Подводные камни

### 1. len считает байты, не символы

```go
len("Привет")  // 12, не 6
```

### 2. Индексация строки даёт байт

```go
s := "Яблоко"
s[0]  // 208 (байт), не "Я"
```

### 3. string(число) интерпретирует код символа

```go
string(rune(72))  // "H", не "72"
```

### 4. Нельзя изменить символ строки

```go
s := "Hello"
// s[0] = 'h'  ❌ строки неизменяемы
// Нужно: через []rune или []byte, изменить, конвертировать обратно
b := []byte(s)
b[0] = 'h'
s = string(b)  // "hello"
```

### 5. Сравнение строк работает напрямую

```go
"abc" == "abc"  // true — можно сравнивать через ==
"abc" < "abd"   // true — лексикографическое сравнение работает
```

> 💡 В отличие от Java, где нужен `.equals()`, в Go строки сравниваются через `==` напрямую. Это удобно.

## 📋 Краткое резюме

- Строка — неизменяемая последовательность **байт** в кодировке UTF-8, не символов.
- `len(string)` возвращает количество байт; для символов — `utf8.RuneCountInString`.
- `s[i]` даёт `byte` — для не-ASCII это кусок символа, а не буква.
- `range` по строке умный: декодирует UTF-8 и выдаёт руны + байтовую позицию.
- Преобразования: `[]byte(s)`, `[]rune(s)` и обратно `string(...)`; у `[]rune` `len` = число рун.
- Ловушка: `string(65)` → `"A"` (код символа), а не `"65"`; для числа в строку — `strconv.Itoa`.
- Сравнение через `'a'` (руна) быстрее, чем `string(r) == "a"` (строка).
- Обычные строки (`"..."`) с escape vs сырые (`` `...` ``) без интерпретации.
