## 📌 Теория

### 2.1 Объявление переменных — два способа

В Go есть два способа объявить переменную. Это не просто синтаксический сахар — у них разные сценарии использования.

#### Способ 1 — `var` (полное объявление)

```go
var name string         // объявление без значения → zero value
var age int = 25        // объявление с типом и значением
var score = 100         // тип выводится автоматически (int)
```

`var` используется:

- На уровне пакета (вне функций)
- Когда нужно явно указать тип
- Когда переменная объявляется сейчас, а значение присвоится позже

#### Способ 2 — `:=` (короткое объявление)

```go
name := "Nikita"        // тип string выводится автоматически
age := 25               // тип int
score := 98.5           // тип float64
```

`:=` используется:

- **Только внутри функций**
- Когда тип очевиден из значения
- Это самый частый способ объявления в Go

> ⚠️ **Важно:** `:=` — это **объявление + присваивание** одновременно. Просто `=` — это только присваивание уже объявленной переменной.

```go
x := 10   // объявляем и присваиваем
x = 20    // просто меняем значение
x := 30   // ❌ ошибка — x уже объявлен в этой области видимости
```

### 2.2 Групповое объявление

```go
// Несколько var
var (
    host    string = "localhost"
    port    int    = 8080
    debug   bool   = true
)

// Несколько := за раз
x, y := 10, 20
name, age := "Nikita", 25

// Множественное присваивание (swap без temp переменной)
x, y = y, x
```

### 2.3 Zero Values — значения по умолчанию

В Go **каждая переменная всегда имеет значение**. Если не инициализировать — получишь zero value для своего типа. Никакого `undefined` как в JS или случайного мусора как в C++.

|Тип|Zero Value|
|---|---|
|`int`, `int8`, `int16`, `int32`, `int64`|`0`|
|`uint`, `uint8`, `uint16`, `uint32`, `uint64`|`0`|
|`float32`, `float64`|`0.0`|
|`complex64`, `complex128`|`0+0i`|
|`bool`|`false`|
|`string`|`""` (пустая строка)|
|`pointer`|`nil`|
|`slice`|`nil`|
|`map`|`nil`|
|`channel`|`nil`|
|`interface`|`nil`|
|`struct`|все поля = их zero values|

```go
var i int       // 0
var f float64   // 0.0
var b bool      // false
var s string    // ""
var p *int      // nil

fmt.Println(i, f, b, s, p)
// 0 0 false  <nil>
```

> 💡 Zero values — это не просто удобство, это **гарантия языка**. Ты всегда знаешь что в переменной, даже если не инициализировал её.

### 2.4 Основные типы данных

#### Целые числа

```go
var a int8   = 127          // -128 до 127
var b int16  = 32767        // -32768 до 32767
var c int32  = 2147483647   // ~±2 млрд
var d int64  = 9223372036854775807

var e uint8  = 255          // 0 до 255 (беззнаковый)
var f uint16 = 65535
var g uint32 = 4294967295
var h uint64 = 18446744073709551615

var i int    = 100          // зависит от платформы: 32 или 64 бит
```

> 💡 В 99% случаев просто используй `int` — он оптимален для платформы. Конкретные размеры нужны при работе с бинарными протоколами, сетью, файлами.

#### Числа с плавающей точкой

```go
var f32 float32 = 3.14      // ~6-7 значимых цифр
var f64 float64 = 3.141592653589793  // ~15-17 значимых цифр
```

> ⚠️ По умолчанию числовой литерал с точкой — это `float64`. Для gamedev это важно: большинство математических операций работают с `float64`.

#### Булевый тип

```go
var isActive bool = true
var isDone bool              // false (zero value)

// В Go нет неявного приведения к bool
if 1 { }        // ❌ ошибка компиляции (в отличие от C/Python)
if true { }     // ✅
```

#### Строки

```go
var s1 string = "Hello"           // обычная строка, двойные кавычки
var s2 string = `Hello\nWorld`    // сырая строка, обратные кавычки — \n не интерпретируется

// Строки в Go — неизменяемые последовательности байт в кодировке UTF-8
s := "Hello"
// s[0] = 'h'  ❌ нельзя изменить отдельный байт
```

> 💡 Подробно строки разберём в уроке 5. Сейчас главное: строки неизменяемы, используют UTF-8.

### 2.5 Псевдонимы типов

В Go есть несколько типов-псевдонимов которые важно знать:

```go
byte    // псевдоним uint8 — используется для работы с байтами/ASCII
rune    // псевдоним int32 — используется для Unicode символов (UTF-8)
```

```go
var b byte = 'A'    // 65
var r rune = 'Я'    // 1071
```

### 2.6 Область видимости переменных

```go
package main

import "fmt"

var globalVar = "я глобальная"  // уровень пакета, видна везде в пакете

func main() {
    localVar := "я локальная"   // видна только в main()

    if true {
        blockVar := "я блочная" // видна только внутри if
        fmt.Println(blockVar)
    }

    // fmt.Println(blockVar) ← ❌ ошибка, blockVar вне области видимости
    fmt.Println(localVar)
    fmt.Println(globalVar)
}
```

> ⚠️ **Shadowing (затенение)** — переменная во внутреннем блоке может "перекрыть" внешнюю:

```go
x := 10
fmt.Println(x) // 10

if true {
    x := 20        // это НОВАЯ переменная x, не присваивание внешней!
    fmt.Println(x) // 20
}

fmt.Println(x)     // 10 — внешняя x не изменилась!
```

> Это частый источник багов. `go vet` и линтеры умеют предупреждать о shadowing.

---

### 2.7 Приведение типов

В Go **нет неявного приведения типов**. Всё нужно делать явно:

```go
var i int = 42
var f float64 = float64(i)   // явное приведение
var u uint = uint(f)

// ❌ Это не скомпилируется:
var x int = 10
var y float64 = x            // cannot use x (type int) as type float64
```

```go
// Пример из gamedev — частая операция:
screenWidth := 1920          // int
ratio := float64(screenWidth) / 16.0  // нужно явно привести к float64
```

---

### 2.8 Константа `_` (blank identifier)

`_` — специальный идентификатор для игнорирования значений. Используется когда функция возвращает несколько значений, но не все нужны:

```go
x, _ := someFunction()  // второе значение игнорируем

// Также используется для импорта ради побочных эффектов
import _ "image/png"    // регистрирует PNG декодер
```

## 💡 Примеры

### Пример 1 — var vs := на практике

```go
package main

import "fmt"

// На уровне пакета — только var
var appName = "MyGame"
var version string  // zero value = ""

func main() {
    // Внутри функции — обычно :=
    playerName := "Hero"
    score := 0
    isAlive := true

    // var когда нужен явный тип
    var damage float32 = 9.5

    fmt.Println(appName, version)
    fmt.Println(playerName, score, isAlive, damage)
}
```

### Пример 2 — Zero values в struct (забегаем вперёд)

```go
package main

import "fmt"

type Player struct {
    Name   string
    Health int
    Score  float64
    IsAlive bool
}

func main() {
    var p Player  // все поля = zero values
    fmt.Println(p)
    // { 0 0 false}
    // Name="" Health=0 Score=0.0 IsAlive=false
}
```

### Пример 3 — Множественное присваивание

```go
package main

import "fmt"

func getCoords() (int, int) {
    return 10, 20
}

func main() {
    x, y := getCoords()
    fmt.Println(x, y) // 10 20

    // Swap без temp
    x, y = y, x
    fmt.Println(x, y) // 20 10

    // Игнорируем второе значение
    x, _ = getCoords()
    fmt.Println(x) // 10
}
```

### Пример 4 — Shadowing ловушка

```go
package main

import "fmt"

func main() {
    err := doSomething()
    if err != nil {
        fmt.Println("ошибка:", err)
    }

    // Частая ловушка с err в if-блоках
    if result, err := riskyOperation(); err != nil {
        // здесь err — это НОВАЯ переменная, только в этом блоке
        fmt.Println("ошибка:", err)
    } else {
        fmt.Println("результат:", result)
    }
}

func doSomething() error { return nil }
func riskyOperation() (int, error) { return 42, nil }
```

## ⚠️ Подводные камни

### 1. `:=` не работает вне функций

```go
package main

name := "test"  // ❌ ошибка компиляции: non-declaration statement outside function body

var name = "test"  // ✅
```

### 2. `:=` требует хотя бы одну новую переменную

```go
x := 10
y := 20
x, y := 30, 40  // ❌ ошибка — оба уже объявлены

x, z := 30, 40  // ✅ — z новая, поэтому можно
```

### 3. Неявное приведение float

```go
x := 5 / 2       // = 2, не 2.5! Это целочисленное деление
y := 5.0 / 2     // = 2.5, потому что 5.0 — это float64
z := 5 / 2.0     // = 2.5
```

### 4. Переполнение без паники

```go
var x int8 = 127
x++
fmt.Println(x)  // -128 — переполнение без ошибки!
```

> ⚠️ Go не паникует при переполнении целых чисел — молча оборачивается. Подробнее в уроке 4.

## 📋 Краткое резюме

- Два способа объявления: `var` (везде, можно без значения) и `:=` (только внутри функций).
- Каждая переменная всегда имеет значение — zero value (`0`, `false`, `""`, `nil`), никакого `undefined`.
- Базовые типы: целые, `float64` по умолчанию для дробных, `bool`, `string`; `byte`=`uint8`, `rune`=`int32`.
- Нет неявного приведения типов — всё через `T(value)`.
- Условие — строго `bool`; `if 1` и `if ""` не скомпилируются.
- Shadowing: `:=` во вложенном блоке создаёт **новую** переменную, не меняет внешнюю.
- `_` (blank identifier) — для игнорирования ненужных значений.
