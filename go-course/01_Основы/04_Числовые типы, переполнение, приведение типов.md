## 📌 Теория

### 4.1 Полная карта числовых типов

Go даёт точный контроль над размером чисел. Это важно для производительности, работы с бинарными данными и gamedev.

#### Целые со знаком (signed)

|Тип|Размер|Диапазон|
|---|---|---|
|`int8`|8 бит|−128 … 127|
|`int16`|16 бит|−32 768 … 32 767|
|`int32`|32 бита|−2 147 483 648 … 2 147 483 647|
|`int64`|64 бита|−9.2×10¹⁸ … 9.2×10¹⁸|
|`int`|32 или 64|зависит от платформы|

#### Целые без знака (unsigned)

|Тип|Размер|Диапазон|
|---|---|---|
|`uint8`|8 бит|0 … 255|
|`uint16`|16 бит|0 … 65 535|
|`uint32`|32 бита|0 … 4 294 967 295|
|`uint64`|64 бита|0 … 1.8×10¹⁹|
|`uint`|32 или 64|зависит от платформы|
|`uintptr`|размер указателя|для хранения адресов (см. урок 51)|

#### Числа с плавающей точкой

|Тип|Размер|Точность|
|---|---|---|
|`float32`|32 бита|~6–7 десятичных цифр|
|`float64`|64 бита|~15–17 десятичных цифр|

#### Комплексные числа

|Тип|Состоит из|
|---|---|
|`complex64`|два `float32`|
|`complex128`|два `float64`|

> 💡 Комплексные числа в Go встроены в язык, но в gamedev почти не используются. Знать что они есть — достаточно.

---

### 4.2 Какой тип выбирать

```go
// По умолчанию:
var count int          // целые — просто int
var ratio float64      // дробные — float64

// Конкретные размеры — когда есть причина:
var pixel uint8        // цвет в RGBA (0-255)
var fileSize int64     // большие размеры файлов
var port uint16        // сетевой порт (0-65535)
var flags uint32       // битовые флаги
```

> 💡 **Правило:** используй `int` и `float64` по умолчанию. Конкретные размеры — только когда работаешь с бинарными форматами, сетью, памятью или нужна экономия (большие массивы).

### 4.3 Переполнение (overflow)

Это критически важная тема. В Go целочисленное переполнение **не вызывает ошибку** — число молча "оборачивается".

```go
var x uint8 = 255
x++
fmt.Println(x)  // 0 — обернулось! (255 + 1 = 0)

var y int8 = 127
y++
fmt.Println(y)  // -128 — обернулось в минимум
```

Почему так? Числа хранятся в фиксированном количестве бит. Когда значение превышает максимум, старшие биты отбрасываются:

```
uint8: 255 = 11111111
       +1
       = 100000000 (9 бит)
       старший бит отброшен → 00000000 = 0
```

> ⚠️ Это поведение унаследовано от C и работает так же в большинстве компилируемых языков. В Python таких проблем нет (числа неограниченные), а в Go — есть. Помни об этом.

#### Как защититься

```go
// Проверка перед операцией
var x uint8 = 200
var add uint8 = 100

if x > 255-add {
    fmt.Println("будет переполнение!")
} else {
    x += add
}
```

Для серьёзной работы с риском переполнения есть пакет `math` с константами границ:

```go
import "math"

fmt.Println(math.MaxInt8)   // 127
fmt.Println(math.MinInt8)   // -128
fmt.Println(math.MaxUint8)  // 255
fmt.Println(math.MaxInt64)  // 9223372036854775807
```

### 4.4 Приведение типов (conversion)

В Go нет неявного приведения. Всё явно через `T(value)`:

```go
var i int = 42
var f float64 = float64(i)   // int → float64
var u uint = uint(f)         // float64 → uint (дробная часть отбрасывается!)
var i8 int8 = int8(i)        // int → int8

fmt.Println(i, f, u)
```

#### Потеря данных при приведении

```go
// float → int отбрасывает дробную часть (НЕ округляет!)
var f float64 = 3.99
var i int = int(f)
fmt.Println(i)  // 3, не 4!

// большой тип → маленький может потерять данные
var big int32 = 300
var small int8 = int8(big)
fmt.Println(small)  // 44 (300 % 256, переполнение)
```

> ⚠️ Приведение `float → int` **усекает**, не округляет. Для округления используй `math.Round()`:

```go
import "math"

f := 3.99
rounded := int(math.Round(f))  // 4
truncated := int(f)            // 3
```

### 4.5 Деление и остаток

```go
// Целочисленное деление
fmt.Println(7 / 2)    // 3 (остаток отбрасывается)
fmt.Println(7 % 2)    // 1 (остаток)

// Дробное деление
fmt.Println(7.0 / 2)  // 3.5

// Деление на ноль
var x int = 10
// fmt.Println(x / 0)  ❌ паника в рантайме: integer divide by zero

var f float64 = 10
fmt.Println(f / 0)    // +Inf — float не паникует!
fmt.Println(0.0 / 0)  // NaN
```

> ⚠️ Целочисленное деление на ноль — **паника**. Деление float на ноль — даёт `+Inf`, `-Inf` или `NaN`, без паники.

---

### 4.6 Специальные значения float

```go
import "math"

posInf := math.Inf(1)   // +Inf
negInf := math.Inf(-1)  // -Inf
nan := math.NaN()       // NaN (Not a Number)

// Проверки
fmt.Println(math.IsNaN(nan))      // true
fmt.Println(math.IsInf(posInf, 1)) // true

// ⚠️ NaN не равен ничему, даже самому себе!
fmt.Println(nan == nan)  // false
```

> ⚠️ Сравнение `NaN == NaN` всегда `false`. Чтобы проверить на NaN — только через `math.IsNaN()`.

---

### 4.7 Точность float — классическая ловушка

```go
fmt.Println(0.1 + 0.2)         // 0.30000000000000004
fmt.Println(0.1+0.2 == 0.3)    // false!
```

Это не баг Go — так работают числа с плавающей точкой во всех языках (стандарт IEEE 754). Дробные числа хранятся приблизительно.

#### Как сравнивать float правильно

```go
import "math"

a := 0.1 + 0.2
b := 0.3
epsilon := 1e-9

if math.Abs(a-b) < epsilon {
    fmt.Println("равны (с учётом погрешности)")
}
```

> 💡 Для денег и точных расчётов не используй float! Используй целые числа (храни центы вместо рублей) или пакет `math/big`.

---

### 4.8 Числовые литералы

Go поддерживает разные системы счисления и разделители:

```go
decimal := 1000000
binary  := 0b1010      // 10 в двоичной
octal   := 0o17        // 15 в восьмеричной
hex     := 0xFF        // 255 в шестнадцатеричной

// Разделитель _ для читаемости (игнорируется компилятором)
million := 1_000_000
bytes   := 0b1010_1010

// Float и научная нотация
f1 := 3.14
f2 := 1.5e3   // 1500
f3 := 2.5e-4  // 0.00025
```

## 💡 Примеры

### Пример 1 — Демонстрация переполнения

```go
package main

import (
    "fmt"
    "math"
)

func main() {
    var x uint8 = math.MaxUint8  // 255
    fmt.Println("До:", x)
    x++
    fmt.Println("После переполнения:", x)  // 0

    var y int8 = math.MaxInt8  // 127
    fmt.Println("До:", y)
    y++
    fmt.Println("После переполнения:", y)  // -128
}
```

### Пример 2 — Безопасное приведение с округлением

```go
package main

import (
    "fmt"
    "math"
)

func main() {
    v := 3.5

    truncated := int(v)            // усечение
    rounded := int(math.Round(v))  // округление

    fmt.Printf("%.1f → усечение: %d, округление: %d\n", v, truncated, rounded)
    // 3.5 → усечение: 3, округление: 4

    v = 3.2
    fmt.Printf("%.1f → усечение: %d, округление: %d\n", v, int(v), int(math.Round(v)))
    // 3.2 → усечение: 3, округление: 3
}
```

### Пример 3 — Сравнение float с погрешностью

```go
package main

import (
    "fmt"
    "math"
)

func floatEquals(a, b float64) bool {
    const epsilon = 1e-9
    return math.Abs(a-b) < epsilon
}

func main() {
    a := 0.1 + 0.2
    b := 0.3

    fmt.Println("Прямое сравнение:", a == b)        // false
    fmt.Println("С погрешностью:", floatEquals(a, b)) // true
}
```

### Пример 4 — RGBA цвет через uint8 (gamedev)

> 💡 Здесь используется `struct` — её подробно разберём в уроке 15. Сейчас просто обрати внимание на применение `uint8` для цветовых каналов.

```go
package main

import "fmt"

type Color struct {
    R, G, B, A uint8  // каждый канал 0-255
}

func main() {
    red := Color{R: 255, G: 0, B: 0, A: 255}
    fmt.Printf("Красный: #%02X%02X%02X\n", red.R, red.G, red.B)
    // Красный: #FF0000

    // Попытка превысить 255 поймается на компиляции для литерала:
    // bad := Color{R: 300}  ❌ constant 300 overflows uint8
}
```

## ⚠️ Подводные камни

### 1. Переполнение молчит

```go
var x int8 = 100
x = x + 100  // 200 не влезает в int8 → -56, без ошибки
```

### 2. float → int усекает, не округляет

```go
int(3.99)  // 3, не 4
```

### 3. Целочисленное деление на ноль — паника

```go
x := 0
10 / x  // panic: integer divide by zero
```

### 4. Сравнение float через == ненадёжно

```go
0.1 + 0.2 == 0.3  // false
```

### 5. Нельзя смешивать типы в выражениях

```go
var a int = 10
var b int64 = 20
// c := a + b  ❌ mismatched types int and int64
c := int64(a) + b  // ✅
```

### 6. NaN не равен сам себе

```go
math.NaN() == math.NaN()  // false
```

## 📋 Краткое резюме

- Полный набор числовых типов: `int8..64`, `uint8..64`, `float32/64`; по умолчанию — `int` и `float64`.
- Целочисленное переполнение **не вызывает ошибку** — число молча оборачивается (в отличие от Python).
- Границы типов — в пакете `math` (`math.MaxUint8`, `math.MaxInt64` и т.д.).
- Приведение `float → int` усекает дробную часть, **не округляет**; для округления — `math.Round`.
- Целочисленное деление на ноль — паника; деление float на ноль — `Inf`/`NaN`, без паники.
- Сравнивать float через `==` ненадёжно (`0.1+0.2 != 0.3`) — используй epsilon; `NaN != NaN`.
- Нельзя смешивать разные числовые типы в выражении без явного приведения.
