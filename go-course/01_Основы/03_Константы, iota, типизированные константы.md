## 📌 Теория

### 3.1 Что такое константа

Константа — это значение которое **определяется на этапе компиляции** и не может быть изменено. В Go константы объявляются через `const`.

```go
const Pi = 3.14159
const AppName = "MyGame"
const MaxPlayers = 4

// Pi = 3.14  ❌ ошибка компиляции — нельзя менять константу
```

> 💡 Ключевое отличие от переменной: константа существует только во время компиляции. Её значение "вшивается" в код. Поэтому константой не может быть то, что вычисляется во время выполнения (например, результат функции или текущее время).

```go
const x = 10           // ✅
const y = 10 * 60      // ✅ вычисляется на этапе компиляции
const z = someFunc()   // ❌ функция вызывается в рантайме
const t = len("hello") // ✅ len от строкового литерала — компайл-тайм
```

### 3.2 Типизированные и нетипизированные константы

Это одна из самых интересных особенностей Go. Константы бывают двух видов.

#### Нетипизированные константы (untyped)

```go
const Pi = 3.14159   // нетипизированная — "просто число"
```

Нетипизированная константа не имеет конкретного типа до момента использования. Она получает тип в контексте:

```go
const Pi = 3.14159

var f float64 = Pi   // здесь Pi становится float64
var f32 float32 = Pi // здесь Pi становится float32

// Это работает! Одна константа подстроилась под разные типы
```

#### Типизированные константы (typed)

```go
const Pi float64 = 3.14159  // типизированная — строго float64

var f32 float32 = Pi  // ❌ ошибка! Pi это float64, нельзя присвоить float32
```

> ⚠️ **Важный нюанс:** нетипизированные константы гибче. Они автоматически конвертируются под нужный тип. Это позволяет писать `const MaxSize = 100` и использовать это значение и как `int`, и как `float64`, и как `int64`.

---

### 3.3 Почему это важно

```go
const timeout = 30  // нетипизированная

var seconds int = timeout       // ок
var millis int64 = timeout * 1000  // ок
var ratio float64 = timeout / 2.0  // ок

// Если бы было const timeout int = 30, то последняя строка
// потребовала бы явного приведения float64(timeout)
```

Нетипизированные константы позволяют избежать "приведенческого ада". Поэтому **по умолчанию объявляй константы без типа**, если нет конкретной причины указать тип.

### 3.5 iota — генератор последовательностей

`iota` — это специальный идентификатор для создания последовательных значений в блоке `const`. Начинается с `0` и увеличивается на `1` с каждой строкой блока.

```go
const (
    A = iota  // 0
    B = iota  // 1
    C = iota  // 2
)
```

Но повторять `= iota` не нужно — он применяется автоматически к каждой строке:

```go
const (
    A = iota  // 0
    B         // 1
    C         // 2
    D         // 3
)
```

> 💡 Это аналог `enum` из C#/Kotlin/C++, которого в Go formально нет. `iota` — идиоматичный способ создавать перечисления.

---

### 3.6 iota — продвинутые приёмы

#### Пропуск значений через `_`

```go
const (
    _  = iota  // 0 — пропускаем
    KB = 1 << (10 * iota)  // 1 << 10 = 1024
    MB                     // 1 << 20 = 1048576
    GB                     // 1 << 30
    TB                     // 1 << 40
)
```

#### iota с выражениями

```go
const (
    _           = iota             // 0, игнорируем
    Low         = iota * 10        // 10
    Medium                         // 20
    High                           // 30
)
```

#### Сброс iota в новом блоке

```go
const (
    A = iota  // 0
    B         // 1
)

const (
    C = iota  // 0 — iota сбрасывается в каждом новом const блоке
    D         // 1
)
```

#### Типизированный enum через iota

```go
type Direction int

const (
    North Direction = iota  // 0
    East                    // 1
    South                   // 2
    West                    // 3
)
```

Теперь `Direction` — это полноценный тип, можно создавать методы:

```go
func (d Direction) String() string {
    switch d {
    case North: return "North"
    case East:  return "East"
    case South: return "South"
    case West:  return "West"
    }
    return "Unknown"
}
```

### 3.7 Битовые флаги через iota

Очень частый паттерн в gamedev — флаги состояний через битовые сдвиги:

```go
type State uint8

const (
    StateIdle    State = 1 << iota  // 1  (00000001)
    StateMoving                     // 2  (00000010)
    StateJumping                    // 4  (00000100)
    StateAttacking                  // 8  (00001000)
)

// Комбинирование флагов
playerState := StateMoving | StateAttacking  // 2 | 8 = 10

// Проверка флага
if playerState & StateMoving != 0 {
    fmt.Println("игрок движется")
}

// Снятие флага
playerState &^= StateMoving  // убрать StateMoving
```

## 💡 Примеры

### Пример 1 — Нетипизированные vs типизированные

```go
package main

import "fmt"

const untypedPi = 3.14159        // нетипизированная
const typedPi float64 = 3.14159  // типизированная

func main() {
    var f32 float32 = untypedPi   // ✅ работает
    // var f32b float32 = typedPi  // ❌ не скомпилируется

    fmt.Println(f32)
}
```

### Пример 2 — Размеры файлов через iota

```go
package main

import "fmt"

const (
    _  = iota
    KB = 1 << (10 * iota)
    MB
    GB
)

func main() {
    fmt.Println("KB:", KB)  // 1024
    fmt.Println("MB:", MB)  // 1048576
    fmt.Println("GB:", GB)  // 1073741824
}
```

### Пример 3 — Enum уровней сложности

```go
package main

import "fmt"

type Difficulty int

const (
    Easy Difficulty = iota
    Normal
    Hard
    Nightmare
)

func (d Difficulty) String() string {
    return [...]string{"Easy", "Normal", "Hard", "Nightmare"}[d]
}

func main() {
    current := Hard
    fmt.Println("Сложность:", current)  // Сложность: Hard
    fmt.Println("Номер:", int(current)) // Номер: 2
}
```

### Пример 4 — Битовые флаги разрешений

```go
package main

import "fmt"

type Permission uint8

const (
    Read Permission = 1 << iota  // 1
    Write                        // 2
    Execute                      // 4
)

func main() {
    // Даём права на чтение и запись
    perms := Read | Write  // 3

    fmt.Printf("Permissions: %03b\n", perms)  // 011

    // Проверяем
    if perms&Read != 0 {
        fmt.Println("можно читать")
    }
    if perms&Execute != 0 {
        fmt.Println("можно выполнять")
    } else {
        fmt.Println("нельзя выполнять")
    }
}
```

## ⚠️ Подводные камни

### 1. Константой не может быть результат рантайм-вычисления

```go
const now = time.Now()       // ❌ функция в рантайме
const size = len(os.Args)    // ❌ зависит от запуска
const x = 10                 // ✅
const y = len("hello")       // ✅ литерал известен на компиляции
```

### 2. iota считает ВСЕ строки, даже пропущенные

```go
const (
    A = iota  // 0
    _         // 1 — iota увеличился, хоть значение и проигнорировано
    C         // 2
)
// C == 2, не 1!
```

### 3. iota только в const, не в var

```go
var x = iota  // ❌ iota работает только внутри const блока
```

### 4. Переполнение типизированной константы ловится на компиляции

```go
const big int8 = 1000  // ❌ ошибка компиляции: overflows int8

const big = 1000       // ✅ нетипизированная, пока не используется — ок
var x int8 = big       // ❌ вот здесь поймает overflow
```

> 💡 Это плюс типизированных констант — ошибки переполнения видны сразу на компиляции, а не в рантайме.

## 📋 Краткое резюме

- Константа вычисляется на этапе компиляции, не может содержать результат рантайм-функции.
- Нетипизированные константы гибче — подстраиваются под контекст, избегают приведений; типизируй только при необходимости.
- `iota` — генератор последовательностей в `const`: стартует с 0, считает все строки блока (даже пропущенные), сбрасывается в новом блоке.
- `iota` — идиоматичная замена enum, которого в Go нет; часто с типом и методом `String()`.
- Битовые флаги через `1 << iota`; комбинирование `|`, проверка `&`, снятие `&^`.
- Степени двойки удобно через `1 << (10*iota)`, но для произвольных степеней (1000) `iota` не подходит — нужна обычная арифметика.
