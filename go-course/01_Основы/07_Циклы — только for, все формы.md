# Урок 07 — Циклы: только for, все формы

**Блок:** [[Go — План курса#📁 Блок 01 — Основы языка (~55 ч)|01 Основы языка]] **Время:** 2–3 часа **Теги:** #go/основы #go/управление

## 📌 Теория

### 7.1 Главная идея — в Go только for

В Go **один-единственный** цикл — `for`. Нет `while`, нет `do-while`, нет `foreach` как отдельных конструкций. Всё это выражается через `for` в разных формах.

> 💡 Это сознательное упрощение в духе Go: вместо пяти ключевых слов для циклов — одно, которое покрывает все случаи. Поначалу непривычно после C#/JS, но на практике очень удобно.

---

### 7.2 Форма 1 — классический for (как в C)

```go
for i := 0; i < 10; i++ {
    fmt.Println(i)
}
```

Три части, разделённые `;`:

1. **инициализация** — `i := 0` (выполняется один раз)
2. **условие** — `i < 10` (проверяется перед каждой итерацией)
3. **пост-инструкция** — `i++` (выполняется после каждой итерации)

> ⚠️ Как и в `if` — **скобок `()` вокруг нет**, фигурные `{}` обязательны.

---

### 7.3 Форма 2 — for как while

Убираешь инициализацию и пост-инструкцию — получаешь аналог `while`:

```go
// Аналог while (x < 100)
x := 1
for x < 100 {
    x *= 2
}
fmt.Println(x)  // 128
```

> 💡 В уроках 1-6 ты уже использовал эту форму (`for cnt < 10`). Теперь знаешь, что это `for` в роли `while`.

---

### 7.4 Форма 3 — бесконечный цикл

```go
for {
    // выполняется вечно, пока не выйдем через break/return
}
```

Это идиоматичный бесконечный цикл. Используется для серверов, игровых циклов, обработки событий:

```go
for {
    cmd := readCommand()
    if cmd == "quit" {
        break
    }
    process(cmd)
}
```

> 💡 В gamedev главный игровой цикл (game loop) — это обычно `for {}` с условием выхода внутри.

---

### 7.5 Форма 4 — for range

`range` перебирает элементы коллекции. Работает со строками, срезами, массивами, картами, каналами.

```go
// По строке — выдаёт (индекс байта, руну)
for i, r := range "Go" {
    fmt.Printf("%d: %c\n", i, r)
}

// По срезу/массиву — выдаёт (индекс, значение)
nums := []int{10, 20, 30}  // срез — подробно в уроке 13
for i, v := range nums {
    fmt.Printf("nums[%d] = %d\n", i, v)
}

// По карте — выдаёт (ключ, значение)
m := map[string]int{"a": 1, "b": 2}  // карта — подробно в уроке 14
for key, value := range m {
    fmt.Printf("%s = %d\n", key, value)
}
```

> 💡 Срезы и карты будем подробно изучать в уроках 13-14. Здесь `range` показан для полноты картины форм `for`. В задачах этого урока сосредоточимся на формах 1-3 и range по строке.

---

### 7.6 break и continue

```go
for i := 0; i < 10; i++ {
    if i == 3 {
        continue  // пропустить эту итерацию, перейти к следующей
    }
    if i == 7 {
        break     // выйти из цикла полностью
    }
    fmt.Println(i)  // напечатает 0,1,2,4,5,6
}
```

---

### 7.7 Метки (labels) — для вложенных циклов

`break` и `continue` по умолчанию работают с **ближайшим** циклом. Для управления внешним циклом используются метки:

```go
outer:
for i := 0; i < 3; i++ {
    for j := 0; j < 3; j++ {
        if i*j > 2 {
            break outer  // выходим сразу из ОБОИХ циклов
        }
        fmt.Println(i, j)
    }
}
```

> ⚠️ Метки используются редко и считаются "запахом кода" если их много. Но для выхода из вложенных циклов это чистое идиоматичное решение в Go (в отличие от флагов-переменных).

---

### 7.8 Игнорирование значений в range

```go
nums := []int{10, 20, 30}

// Нужен только индекс
for i := range nums {
    fmt.Println(i)  // 0, 1, 2
}

// Нужно только значение — индекс игнорируем через _
for _, v := range nums {
    fmt.Println(v)  // 10, 20, 30
}

// Просто N повторений (range по числу — с Go 1.22+)
for i := range 5 {
    fmt.Println(i)  // 0,1,2,3,4
}
```

> 💡 **Новинка Go 1.22:** `for i := range 5` — перебор от 0 до 4. Раньше писали `for i := 0; i < 5; i++`. Проверь свою версию Go — если 1.22+, можешь использовать.

---

### 7.9 Ловушка: переменная цикла (до Go 1.22)

Исторически в Go была знаменитая ловушка с переменной цикла в горутинах и замыканиях:

```go
// До Go 1.22 это печатало 3,3,3 вместо 0,1,2
for i := 0; i < 3; i++ {
    defer func() { fmt.Println(i) }()
}
```

> 💡 **Хорошая новость:** в Go 1.22 это исправили — теперь каждая итерация получает свою копию переменной. Но если работаешь со старым кодом или версией < 1.22 — помни об этой ловушке. Подробно разберём в блоке concurrency (урок 31+).

## 💡 Примеры

### Пример 1 — Все формы for рядом

```go
package main

import "fmt"

func main() {
    // Классический
    fmt.Println("--- классический ---")
    for i := 0; i < 3; i++ {
        fmt.Println(i)
    }

    // Как while
    fmt.Println("--- как while ---")
    n := 1
    for n < 10 {
        n *= 2
    }
    fmt.Println(n)

    // Бесконечный с break
    fmt.Println("--- бесконечный ---")
    count := 0
    for {
        if count >= 3 {
            break
        }
        fmt.Println("tick", count)
        count++
    }
}
```

### Пример 2 — Сумма цифр числа

```go
package main

import "fmt"

func sumDigits(n int) int {
    sum := 0
    for n > 0 {
        sum += n % 10  // берём последнюю цифру
        n /= 10        // отбрасываем последнюю цифру
    }
    return sum
}

func main() {
    fmt.Println(sumDigits(12345))  // 15
    fmt.Println(sumDigits(999))    // 27
}
```

### Пример 3 — Таблица умножения (вложенные циклы)

```go
package main

import "fmt"

func main() {
    for i := 1; i <= 3; i++ {
        for j := 1; j <= 3; j++ {
            fmt.Printf("%d×%d=%d\t", i, j, i*j)
        }
        fmt.Println()  // перенос строки после каждого ряда
    }
}
```

### Пример 4 — Проверка простого числа

```go
package main

import "fmt"

func isPrime(n int) bool {
    if n < 2 {
        return false
    }
    for i := 2; i*i <= n; i++ {  // достаточно проверять до √n
        if n%i == 0 {
            return false
        }
    }
    return true
}

func main() {
    for n := 2; n <= 20; n++ {
        if isPrime(n) {
            fmt.Printf("%d простое\n", n)
        }
    }
}
```

## ⚠️ Подводные камни

### 1. Нет while/do-while

```go
while x < 10 { }  // ❌ нет такого ключевого слова
for x < 10 { }    // ✅ for в роли while
```

### 2. Скобки вокруг условия не нужны

```go
for (i := 0; i < 10; i++) { }  // ❌
for i := 0; i < 10; i++ { }    // ✅
```

### 3. break выходит только из ближайшего цикла

```go
for i := 0; i < 3; i++ {
    for j := 0; j < 3; j++ {
        break  // выходит только из внутреннего цикла j
    }
}
```

### 4. Переменная цикла видна только внутри

```go
for i := 0; i < 10; i++ {
}
// fmt.Println(i)  ❌ i вне области видимости
```

### 5. Бесконечный цикл без выхода

```go
for {
    // если забыть break/return — программа зависнет
}
```

## 📋 Краткое резюме

- В Go единственный цикл — `for`; `while`/`do-while`/`foreach` нет.
- Четыре формы: классическая `for i:=0; i<n; i++`, как-while `for cond`, бесконечная `for {}`, и `for range`.
- `for {}` — идиоматичный бесконечный цикл (основа game loop).
- `break`/`continue` работают с ближайшим циклом; для внешнего — метки (labels).
- `range` по строке даёт руны; по срезу/массиву — индекс+значение; по карте — ключ+значение.
- Go 1.22+: `for i := range N` для N повторений; ловушка переменной цикла исправлена.
- Скобок `()` вокруг условия нет, фигурные `{}` обязательны.
