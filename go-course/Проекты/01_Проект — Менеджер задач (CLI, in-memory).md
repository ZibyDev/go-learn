> 🎯 **Цель:** объединить весь блок 01 в одном приложении. Это твой первый полноценный Go-проект — консольный менеджер задач с интерактивным меню и сохранением в файл.

---

## 📌 Что ты построишь

Консольное приложение для управления списком задач (todo-лист). Пользователь работает через **меню в цикле**: вводит номер команды, программа выполняет действие, снова показывает меню. Данные хранятся в памяти во время работы и **сохраняются в файл** между запусками.

---

## ✅ Функциональные требования

Приложение должно уметь:

1. **Добавить задачу** — пользователь вводит название, приоритет и (опционально) описание. Задача получает уникальный ID и статус «не выполнена».
2. **Показать все задачи** — вывести список с ID, статусом, приоритетом, названием. Отформатировано в читаемую таблицу.
3. **Отметить задачу выполненной** — по ID переключить статус.
4. **Удалить задачу** — по ID убрать из списка.
5. **Фильтровать задачи** — показать только выполненные / только невыполненные / по приоритету.
6. **Статистика** — сколько всего задач, сколько выполнено, сколько по каждому приоритету.
7. **Поиск** — найти задачи, в названии которых есть введённая подстрока (без учёта регистра).
8. **Сохранение/загрузка** — при выходе сохранить в файл, при запуске — загрузить.

## 🧱 Подсказки по структуре

> Это направления, не готовый код. Архитектуру продумай сам — в этом ценность проекта.

### Модель данных

Заведи структуру для задачи. Подумай, какие поля нужны:

- уникальный идентификатор
- название
- описание
- приоритет (вспомни `iota` из урока 3 — приоритет это отличный кандидат на типизированный enum со `String()`)
- флаг выполнения
- (опционально) дата создания

```go
type Priority int

const (
    Low Priority = iota
    Medium
    High
)

type Task struct {
    // продумай поля сам
}
```

### Хранилище

Как хранить набор задач? Есть два подхода — подумай, какой удобнее:

- **срез** `[]Task` — простой перебор, но поиск по ID за O(n)
- **карта** `map[int]Task` — быстрый доступ по ID, но порядок итерации случайный (вспомни урок 14)

> 💡 Можно скомбинировать: карта для быстрого доступа + отдельно следить за порядком. Или срез + функция поиска по ID. Реши сам, обоснуй выбор в комментарии.

### Генерация ID

Нужен механизм уникальных ID. Самое простое — счётчик, который растёт с каждой новой задачей. Подумай, где его хранить, чтобы он переживал перезапуск (подсказка: сохранять вместе с задачами).

### Меню в цикле

Основа интерфейса — бесконечный цикл `for` с чтением ввода:

```go
for {
    printMenu()
    choice := readInput()  // прочитать строку, обрезать пробелы
    switch choice {
    case "1":
        // добавить задачу
    case "2":
        // показать все
    // ...
    case "0":
        // сохранить и выйти (return или break)
    default:
        fmt.Println("неизвестная команда")
    }
}
```

### Чтение ввода с клавиатуры

Для интерактивного ввода понадобится `bufio.Scanner` (пакет `bufio` подробно в уроке 41, здесь — как готовый инструмент):

```go
import (
    "bufio"
    "os"
    "strings"
)

var reader = bufio.NewReader(os.Stdin)

func readLine(prompt string) string {
    fmt.Print(prompt)
    text, _ := reader.ReadString('\n')
    return strings.TrimSpace(text)  // убрать перенос строки и пробелы
}
```

> 💡 `strings.TrimSpace` убирает `\n` и пробелы по краям. Для преобразования введённого ID в число — `strconv.Atoi` (помнишь из урока 6).

## 💾 Бонусная часть — сохранение в файл

Это забегание вперёд (пакеты `os`, `encoding/json` из блока 05), но даю как готовый инструмент. Сохранение через JSON — самый простой способ.

> ⚠️ Для JSON поля структуры должны быть **экспортируемыми** (с большой буквы) — иначе json-пакет их не увидит. Вспомни про экспорт из урока 1.

### Сохранение

```go
import (
    "encoding/json"
    "os"
)

func saveToFile(filename string, tasks []Task) error {
    data, err := json.MarshalIndent(tasks, "", "  ")  // в читаемый JSON
    if err != nil {
        return err
    }
    return os.WriteFile(filename, data, 0644)
}
```

### Загрузка

```go
func loadFromFile(filename string) ([]Task, error) {
    data, err := os.ReadFile(filename)
    if err != nil {
        if os.IsNotExist(err) {
            return []Task{}, nil  // файла нет — начинаем с пустого списка
        }
        return nil, err
    }
    var tasks []Task
    err = json.Unmarshal(data, &tasks)
    return tasks, err
}
```

> 💡 Обрати внимание на обработку «файл не существует» — при первом запуске файла ещё нет, это не ошибка, просто начинаем с пустого списка. Это применение паттерна обработки ошибок из уроков 8-9.

## 🚦 Этапы выполнения (рекомендуемый порядок)

Не пиши всё сразу — двигайся итеративно:

1. **Модель + добавление + вывод.** Структура `Task`, срез/карта, добавить задачу, показать все. Меню с двумя командами. Это уже рабочий каркас.
2. **Полный CRUD.** Добавь отметку выполнения, удаление по ID. Обработай несуществующий ID.
3. **Фильтры, поиск, статистика.** Функции, работающие со списком — отличная практика на срезы/карты/циклы.
4. **Приоритет как enum + String().** Преврати приоритет в типизированную константу с методом.
5. **Сохранение/загрузка.** Подключи JSON. Сохраняй при выходе, загружай при старте.
6. **Полировка.** Обработка краёв (пустой список, неверный ввод), форматирование вывода, `gofmt`/`go vet`.
