# Урок 18 — Пустой интерфейс any, type assertion, type switch

## 📌 Теория

### 18.1 Пустой интерфейс — повторение и углубление

Из урока 17 ты помнишь: пустой интерфейс `interface{}` (или `any` с Go 1.18) реализует **любой** тип, потому что у любого типа есть «ноль требуемых методов».

```go
var x any
x = 42           // int
x = "hello"      // string
x = []int{1, 2}  // срез
x = Circle{}     // структура
```

`any` — это просто псевдоним для `interface{}`. Используй `any`, это современнее и читаемее.

> 💡 `any` нужен, когда тип заранее неизвестен: функции вроде `fmt.Println(...any)`, контейнеры для разнородных данных, JSON произвольной структуры. Но за гибкость платишь: теряется типобезопасность, и чтобы что-то сделать со значением — нужно «достать» его конкретный тип.

---

### 18.2 Проблема: как достать значение обратно

Положив значение в `any`, ты «забываешь» его тип. Напрямую с ним почти ничего нельзя сделать:

```go
var x any = 42
// y := x + 1  // ❌ ошибка: any не поддерживает +
```

Компилятор видит только `any`, а не `int`. Чтобы вернуть конкретный тип — нужен **type assertion** (утверждение типа).

---

### 18.3 Type assertion — извлечение конкретного типа

Синтаксис `x.(T)` утверждает: «значение в `x` имеет тип `T`», и извлекает его:

```go
var x any = 42

n := x.(int)     // утверждаем, что x это int
fmt.Println(n + 1)  // 43 — теперь это int, можно складывать
```

> ⚠️ **Опасность:** если тип не совпал — паника!

```go
var x any = "hello"
n := x.(int)     // panic: interface conversion: interface {} is string, not int
```

---

### 18.4 Безопасный type assertion — «comma ok»

Чтобы не паниковать, используй форму с двумя возвращаемыми значениями (как «comma ok» у карт из урока 14):

```go
var x any = "hello"

n, ok := x.(int)
fmt.Println(n, ok)  // 0 false — не int, но без паники

s, ok := x.(string)
fmt.Println(s, ok)  // hello true — это string
```

`ok` — булев флаг успеха. Если тип не совпал, `ok == false`, а значение — zero value типа `T`. Никакой паники.

```go
if n, ok := x.(int); ok {
    fmt.Println("это int:", n)
} else {
    fmt.Println("не int")
}
```

> 💡 Всегда предпочитай форму с `ok`, если не уверен на 100% в типе. Паникующая форма `x.(T)` — только когда несовпадение типа означает баг в программе.

---

### 18.5 Type switch — switch по типу

Когда нужно обработать **много возможных типов**, делать цепочку `if ... ok` неудобно. Для этого есть **type switch** — особый `switch` по типу значения:

```go
func describe(x any) {
    switch v := x.(type) {
    case int:
        fmt.Printf("целое: %d\n", v)
    case string:
        fmt.Printf("строка: %q (длина %d)\n", v, len(v))
    case bool:
        fmt.Printf("булево: %t\n", v)
    case []int:
        fmt.Printf("срез int, длина %d\n", len(v))
    default:
        fmt.Printf("неизвестный тип: %T\n", v)
    }
}
```

Синтаксис `switch v := x.(type)`:

- `x.(type)` — специальная форма, работает **только** внутри switch
- `v` — в каждом `case` имеет конкретный тип этого case
- `default` — для необработанных типов

> 💡 Ты впервые видел упоминание type switch ещё в уроке 6 (с пометкой «подробно в уроке 18»). Вот он. Внутри каждого `case` переменная `v` уже имеет нужный тип — её можно использовать напрямую.

---

### 18.6 Несколько типов в одном case

Как и в обычном switch, можно перечислить типы через запятую. Но тогда `v` имеет тип **исходного интерфейса**, а не конкретный:

```go
func process(x any) {
    switch v := x.(type) {
    case int, int64:
        fmt.Println("какое-то целое:", v)  // v здесь типа any!
    case string:
        fmt.Println("строка длиной", len(v))  // v точно string
    }
}
```

> ⚠️ В `case int, int64` переменная `v` остаётся `any` (потому что компилятор не знает, который именно). Чтобы работать с конкретным типом — выноси в отдельный `case`.

---

### 18.7 Type switch с интерфейсами

В type switch можно проверять не только конкретные типы, но и **реализацию интерфейсов**:

```go
type Stringer interface {
    String() string
}

func check(x any) {
    switch v := x.(type) {
    case Stringer:
        fmt.Println("умеет String():", v.String())
    case error:
        fmt.Println("это ошибка:", v.Error())
    default:
        fmt.Printf("обычное значение: %v\n", v)
    }
}
```

> 💡 Порядок важен: проверки идут сверху вниз, срабатывает первый подходящий case. Если тип реализует несколько интерфейсов — выберется первый совпавший.

---

### 18.8 Когда использовать any — и когда НЕ использовать

`any` — мощный, но опасный инструмент. Правила:

**Уместно:**

- Данные действительно произвольного типа (JSON, конфиги)
- Контейнеры общего назначения (до дженериков был единственный способ)
- Функции форматирования (`fmt.Println`)

**НЕ уместно:**

- Когда тип известен — используй конкретный тип
- Для «универсальных» функций — теперь есть **дженерики** (урок 21), они типобезопасны
- Лишь бы избежать описания типов — это потеря всех преимуществ статической типизации

> 💡 Идиома Go: _«any не несёт информации»_ (`any` says nothing). Каждый `any` — это место, где ты теряешь проверку типов и можешь получить панику в рантайме. С появлением дженериков (урок 21) многие случаи `any` стали не нужны. Используй `any` осознанно, не как «отключатель типов».

## 💡 Примеры

### Пример 1 — Безопасный assertion

```go
package main

import "fmt"

func tryInt(x any) {
    if n, ok := x.(int); ok {
        fmt.Println("int + 100 =", n+100)
    } else {
        fmt.Printf("%v не int (это %T)\n", x, x)
    }
}

func main() {
    tryInt(42)        // int + 100 = 142
    tryInt("hello")   // hello не int (это string)
    tryInt(3.14)      // 3.14 не int (это float64)
}
```

### Пример 2 — Type switch

```go
package main

import "fmt"

func stringify(x any) string {
    switch v := x.(type) {
    case nil:
        return "nil"
    case int:
        return fmt.Sprintf("int(%d)", v)
    case float64:
        return fmt.Sprintf("float(%.2f)", v)
    case string:
        return fmt.Sprintf("str(%q)", v)
    case bool:
        return fmt.Sprintf("bool(%t)", v)
    default:
        return fmt.Sprintf("unknown(%T)", v)
    }
}

func main() {
    fmt.Println(stringify(42))      // int(42)
    fmt.Println(stringify(3.14))    // float(3.14)
    fmt.Println(stringify("hi"))    // str("hi")
    fmt.Println(stringify(true))    // bool(true)
    fmt.Println(stringify(nil))     // nil
    fmt.Println(stringify([]int{})) // unknown([]int)
}
```

### Пример 3 — Обработка разнородного среза

```go
package main

import "fmt"

func main() {
    mixed := []any{42, "hello", 3.14, true, []int{1, 2}}

    for _, item := range mixed {
        switch v := item.(type) {
        case int:
            fmt.Println("число:", v*2)
        case string:
            fmt.Println("текст:", v+"!")
        case float64:
            fmt.Printf("дробь: %.1f\n", v)
        default:
            fmt.Printf("прочее: %v\n", v)
        }
    }
}
```

### Пример 4 — assertion к интерфейсу

```go
package main

import "fmt"

type Stringer interface {
    String() string
}

type Point struct{ X, Y int }

func (p Point) String() string {
    return fmt.Sprintf("(%d, %d)", p.X, p.Y)
}

func printValue(x any) {
    // Проверяем, реализует ли x интерфейс Stringer
    if s, ok := x.(Stringer); ok {
        fmt.Println("Stringer:", s.String())
    } else {
        fmt.Println("обычное:", x)
    }
}

func main() {
    printValue(Point{1, 2})  // Stringer: (1, 2)
    printValue(42)           // обычное: 42
}
```

## ⚠️ Подводные камни

### 1. Прямой assertion паникует при несовпадении

```go
var x any = "hello"
n := x.(int)  // panic! используй n, ok := x.(int)
```

### 2. В case с несколькими типами v остаётся any

```go
switch v := x.(type) {
case int, int64:
    // v здесь any, не int — нельзя v + 1
}
```

### 3. x.(type) работает только в switch

```go
// t := x.(type)  // ❌ ошибка вне switch
switch x.(type) { ... }  // ✅
```

### 4. any прячет ошибки до рантайма

```go
func add(a, b any) any {
    return a.(int) + b.(int)  // паника, если не int — а компилятор не предупредит
}
```

### 5. Сравнение any может паниковать

```go
var a any = []int{1}
var b any = []int{1}
// a == b  // panic: срезы несравнимы (но any это прячет до рантайма)
```

## 📋 Краткое резюме

- `any` = `interface{}` — реализуется любым типом; для произвольных данных.
- Положив значение в `any`, теряешь его конкретный тип — напрямую с ним работать нельзя.
- **Type assertion** `x.(T)` извлекает конкретный тип; прямая форма **паникует** при несовпадении.
- Безопасная форма «comma ok» `v, ok := x.(T)` — без паники, `ok` показывает успех.
- **Type switch** `switch v := x.(type)` — обработка многих типов; `v` в каждом case имеет тип этого case.
- В `case` с несколькими типами `v` остаётся типом интерфейса (`any`).
- Type switch умеет проверять и реализацию интерфейсов, не только конкретные типы.
- `x.(type)` работает только внутри switch.
- `any` says nothing: каждый `any` — потеря типобезопасности и риск паники; с дженериками (урок 21) часто не нужен.
- Используй `any` осознанно: для действительно разнородных данных, не как «отключатель типов».
