### Задача 3 — Структура как ключ

Объяви структуру `Cell{Row, Col int}`. Создай карту `map[Cell]string` (например, шахматная доска: клетка → название фигуры). Заполни несколько клеток, проверь наличие фигуры на разных клетках через comma ok. Объясни в комментарии, почему `Cell` можно использовать как ключ, а структуру со срезом — нельзя.

```go
package main

import "fmt"

type Cell struct {
	Row, Col int
}

func main() {
	chess := map[Cell]string{
		{Row: 1, Col: 1}: "пешка",
		{Row: 4, Col: 3}: "конь",
		{Row: 2, Col: 1}: "ладья",
		{Row: 6, Col: 8}: "король",
	}
	for i := 1; i <= 8; i++ {
		for j := 1; j <= 8; j++ {
			if v, ok := chess[Cell{Row: i, Col: j}]; ok {
				fmt.Println(i, j, v)
			}
		}
	}
}
```
Вывод:
```
1 1 пешка
2 1 ладья
4 3 конь
6 8 король
```
Комментарий:
структуру можно использовать как ключ, потому что она работает как массив (с оговорками, её можно изменять, но она имеет стабильную длину в отличие от среза). При этом если в нашей структуре будет срез, её нельзя будет использовать в мапе, так как у срезов много параметров для сравнения и их сравнение в обычном go приводил к ошибке компиляции
